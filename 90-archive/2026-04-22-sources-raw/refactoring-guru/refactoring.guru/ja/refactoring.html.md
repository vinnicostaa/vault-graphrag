# Refactoring {#refactoring .ref1-h .ref-abs .ref-c .ref-h1}

::: mob-image
[![Refactoring](../images/refactoring/content/index/refactoring-i166e6.png?id=4ba85db5ddb3da0cd829bc7c0d2bf407){width="256"
height="192"
srcset="/images/refactoring/content/index/refactoring-i1-2x.png?id=dc9209e9ccee993f51c33697c716aa49 2x,/images/refactoring/content/index/refactoring-i1-3x.png 3x"}](../refactoring/what-is-refactoring.html "Intro to refactoring")
:::

Refactoring is a systematic process of improving code without creating
new functionality that can transform a mess into clean code and simple
design.

beginning](../refactoring/what-is-refactoring.html){.btn .btn-primary

###  Clean Code {#clean-code .ref3-h .ref-abs .ref-r .ref-h3}

::: mob-image
[![Clean
Code](../images/refactoring/content/index/refactoring-i3179f.png?id=e393546f4f2d0046fdbf98b75c6c35ea){width="256"

###  Code Smells {#code-smells .ref5-h .ref-abs .ref-l .ref-h3}

::: mob-image
[![Code
Smells](../images/refactoring/content/index/refactoring-i5a1cd.png?id=acb5ca2349f078fa48ea57b5ad1ee0f5){width="256"
height="192" loading="lazy"
srcset="/images/refactoring/content/index/refactoring-i5-2x.png?id=81451a200d668d4ec23de6a80b3efc80 2x,/images/refactoring/content/index/refactoring-i5-3x.png 3x"}](../refactoring/smells.html "Code Smells")

Code smells are indicators of problems that can be addressed during
refactoring. Code smells are easy to spot and fix, but they may be just
