# Раздувальщики {#раздувальщики .title}

Раздувальщики представляют код, методы и классы, которые раздулись до
таких больших размеров, что с ними стало невозможно эффективно работать.
Все эти запахи зачастую не появляются сразу, а нарастают в процессе
эволюции программы (особенно когда никто не пытается бороться с ними).

:::::::::::::::::: {.relations .link-list}
::::: dl
::: dt
[Long Method](../../smells/long-method.html)
:::

::: dd
Метод содержит слишком большое число строк кода. Длина метода более
десяти строк должна начинать вас беспокоить.
:::
:::::

::::: dl
::: dt
[Large Class](../../smells/large-class.html)
:::

::: dd
Класс содержит множество полей/методов/строк кода.
:::
:::::

::::: dl
::: dt
[Primitive Obsession](../../smells/primitive-obsession.html)
:::

::: dd
-   Использование элементарных типов вместо маленьких объектов для
    небольших задач (например, валюта, диапазоны, специальные строки для
    телефонных номеров и т. п.)
-   Использование констант для кодирования какой-то информации
    (например, константа `USER_ADMIN_ROLE = 1` для обозначения
    пользователей с ролью администратора).
-   Использование строковых констант в качестве названий полей в
    массивах.
:::
:::::

::::: dl
::: dt
[Long Parameter List](../../smells/long-parameter-list.html)
:::

::: dd
Количество параметров метода больше трёх-четырёх.
:::

::::: dl

[Data Clumps](../../smells/data-clumps.html)
:::

::: dd

(например, параметры подключения к базе данных). Такие группы следует
превращать в самостоятельные классы.
