# Упрощение вызовов методов {#упрощение-вызовов-методов .title}

Эти рефакторинги делают вызовы методов проще и яснее для понимания. Это,
в свою очередь, упрощает интерфейсы взаимодействия между классами.

::::::::::::::::::::::::::::::::::::::::::::: {.relations .link-list}
::::: dl
::: dt
[Rename Method](../../rename-method.html)
:::

::: dd
**Проблема:** Название метода не раскрывает суть того, что он делает.

**Решение:** Измените название метода.
:::
:::::

::::: dl
::: dt
[Add Parameter](../../add-parameter.html)
:::

::: dd
**Проблема:** Методу не хватает данных для осуществления каких-то
действий.

**Решение:** Создайте новый параметр, чтобы передать эти данные.
:::
:::::

::::: dl
::: dt
[Remove Parameter](../../remove-parameter.html)
:::

::: dd
**Проблема:** Параметр не используется в теле метода.

**Решение:** Удалите неиспользуемый параметр.
:::
:::::

::::: dl
::: dt
[Separate Query from Modifier](../../separate-query-from-modifier.html)
:::

::: dd
**Проблема:** У вас есть метод, который возвращает какое-то значение, но
при этом в процессе работы он изменяет что-то внутри объекта.

**Решение:** Разделите метод на два разных метода. Один из них пускай
возвращает значение, а второй модифицирует объект.
:::
:::::

::::: dl
::: dt
[Parameterize Method](../../parameterize-method.html)
:::

::: dd
**Проблема:** Несколько методов выполняют похожие действия, которые
отличаются только какими-то внутренними значениями, числами или
операциями.

**Решение:** Объедините все эти методы в один с параметром, в который
будет передаваться отличающееся значение.
:::
:::::

::::: dl
::: dt
[Replace Parameter with Explicit
Methods](../../replace-parameter-with-explicit-methods.html)
:::

::: dd
**Проблема:** Метод разбит на части, каждая из которых выполняется в
зависимости от значения какого-то параметра.

**Решение:** Извлеките отдельные части метода в собственные методы и
вызывайте их вместо оригинального метода.
:::
:::::

::::: dl
::: dt
[Preserve Whole Object](../../preserve-whole-object.html)
:::

::: dd
**Проблема:** Вы получаете несколько значений из объекта, а затем
передаёте их в метод как параметры.

**Решение:** Вместо этого передавайте весь объект.
:::
:::::

::::: dl
::: dt
[Replace Parameter with Method
Call](../../replace-parameter-with-method-call.html)
:::

::: dd
**Проблема:** Вызываем метод и передаем его результаты как параметры
другого метода. При этом значение параметров могли бы быть получены и
внутри вызываемого метода.

**Решение:** Вместо передачи значения через параметры метода, попробуйте
переместить код получения значения внутрь самого метода.
:::
:::::

::::: dl
::: dt
[Introduce Parameter Object](../../introduce-parameter-object.html)
:::

::: dd
**Проблема:** В ваших методах встречается повторяющаяся группа
параметров.

**Решение:** Замените эти параметры объектом.
:::
:::::

::::: dl
::: dt
[Remove Setting Method](../../remove-setting-method.html)
:::

::: dd
**Проблема:** Значение поля должно быть установлено только в момент
создания и больше никогда не меняться.

**Решение:** Удалите методы, устанавливающие значение этого поля.
:::
:::::

::::: dl
::: dt
[Hide Method](../../hide-method.html)
:::

::: dd
**Проблема:** Метод не используется другими классами либо используется
только внутри своей иерархии классов.

**Решение:** Сделайте метод приватным или защищённым.
:::
:::::

::::: dl
::: dt
[Replace Constructor with Factory
Method](../../replace-constructor-with-factory-method.html)
:::

::: dd
**Проблема:** У вас есть сложный конструктор, делающий нечто большее,
чем простая установка значений полей объекта.

**Решение:** Создайте фабричный метод и замените им вызовы конструктора.
:::
:::::

::::: dl
::: dt
[Replace Error Code with
Exception](../../replace-error-code-with-exception.html)
:::

::: dd
**Проблема:** Метод возвращает определенное значение, которое будет
сигнализировать об ошибке.

**Решение:** Вместо этого следует выбрасывать исключение.
:::
:::::

::::: dl
::: dt
[Replace Exception with Test](../../replace-exception-with-test.html)
:::

::: dd
**Проблема:** Вы выбрасываете исключение там, где можно было бы обойтись
простой проверкой условия.

**Решение:** Замените выбрасывание исключения проверкой этого условия.
:::
