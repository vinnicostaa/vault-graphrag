# Организация данных {#организация-данных .title}

Рефакторинги этой группы призваны облегчить работу с данными, заменив
работу с примитивными типами богатыми функциональностью классами.

Кроме того, важным моментом является уменьшение связанности между
классами, что улучшает переносимость классов и шансы их повторного
использования.

:::::::::::::::::::::::::::::::::::::::::::::::: {.relations .link-list}
::::: dl
::: dt
[Self Encapsulate Field](../../self-encapsulate-field.html)
:::

::: dd
**Проблема:** Вы используете прямой доступ к приватным полями внутри
класса.

**Решение:** Создайте геттер и сеттер для поля, и пользуйтесь для
доступа к полю только ими.
:::
:::::

::::: dl
::: dt
[Replace Data Value with
Object](../../replace-data-value-with-object.html)
:::

::: dd
**Проблема:** В классе (или группе классов) есть поле простого типа. У
этого поля есть своё поведение и связанные данные.

**Решение:** Создайте новый класс, поместите в него старое поле и его
поведения, храните объект этого класса в исходном классе.
:::
:::::

::::: dl
::: dt
[Change Value to Reference](../../change-value-to-reference.html)
:::

::: dd
**Проблема:** Есть много одинаковых экземпляров одного класса, которые
можно заменить одним объектом.

**Решение:** Превратите одинаковые объекты в один объект-ссылку.
:::
:::::

::::: dl
::: dt
[Change Reference to Value](../../change-reference-to-value.html)
:::

::: dd
**Проблема:** У вас есть объект-ссылка, который слишком маленький и
неизменяемый, чтобы оправдать сложности по управлению его жизненным
циклом.

**Решение:** Превратите его в объект-значение.
:::
:::::

::::: dl
::: dt
[Replace Array with Object](../../replace-array-with-object.html)
:::

::: dd
**Проблема:** У вас есть массив, в котором хранятся разнотипные данные.

**Решение:** Замените массив объектом, который будет иметь отдельные
поля для каждого элемента.
:::
:::::

::::: dl
::: dt
[Duplicate Observed Data](../../duplicate-observed-data.html)
:::

::: dd
**Проблема:** Данные предметной области программы хранятся в классах,
отвечающих за пользовательский интерфейс (GUI).

**Решение:** Имеет смысл выделить данные предметной области в отдельные
классы и, таким образом, обеспечить связь и синхронизацию между классом
предметной области и GUI.
:::
:::::

::::: dl
::: dt
[Change Unidirectional Association to
Bidirectional](../../change-unidirectional-association-to-bidirectional.html)
:::

::: dd
**Проблема:** У вас есть два класса, которым нужно использовать фичи
друг друга, но между ними существует только односторонняя связь.

**Решение:** Добавьте недостающую связь в класс, в котором она
отсутствует.
:::
:::::

::::: dl
::: dt
[Change Bidirectional Association to
Unidirectional](../../change-bidirectional-association-to-unidirectional.html)
:::

::: dd
**Проблема:** У вас есть двухсторонняя связь между классами, но один из
классов больше не использует фичи другого.

**Решение:** Уберите неиспользуемую связь.
:::
:::::

::::: dl
::: dt
[Replace Magic Number with Symbolic
Constant](../../replace-magic-number-with-symbolic-constant.html)
:::

::: dd
**Проблема:** В коде используется число, которое несёт какой-то
определённый смысл.

**Решение:** Замените это число константой с человеко-читаемым
названием, объясняющим смысл этого числа.
:::
:::::

::::: dl
::: dt
[Encapsulate Field](../../encapsulate-field.html)
:::

::: dd
**Проблема:** У вас есть публичное поле.

**Решение:** Сделайте поле приватным и создайте для него методы доступа.
:::
:::::

::::: dl
::: dt
[Encapsulate Collection](../../encapsulate-collection.html)
:::

::: dd
**Проблема:** Класс содержит поле-коллекцию и простой геттер и сеттер
для работы с этой коллекцией.

**Решение:** Сделайте возвращаемое геттером значение доступным только
для чтения и создайте методы добавления/удаления элементов этой
коллекции.
:::
:::::

::::: dl
::: dt
[Replace Type Code with Class](../../replace-type-code-with-class.html)
:::

::: dd
**Проблема:** В классе есть поле, содержащее кодирование типа. Значения
этого типа не используются в условных операторах и не влияют на
поведение программы.

**Решение:** Создайте новый класс и применяйте его объекты вместо
значений закодированного типа.
:::
:::::

::::: dl
::: dt
[Replace Type Code with
Subclasses](../../replace-type-code-with-subclasses.html)
:::

::: dd
**Проблема:** У вас есть закодированный тип, который непосредственно
влияет на поведение программы (основываясь на значениях этого поля, в
условных операторах выполняется различный код).

**Решение:** Для каждого значения закодированного типа, создайте
подклассы. А затем, вынесите соответствующие поведения из исходного
класса в эти подклассы. Управляющий код замените полиморфизмом.
:::
:::::

::::: dl
::: dt
[Replace Type Code with
State/Strategy](../../replace-type-code-with-state-strategy.html)
:::

::: dd
**Проблема:** У вас есть закодированный тип, который влияет на
поведение, но вы не можете использовать подклассы, чтобы избавиться от
него.

**Решение:** Замените кодирование типа объектом-состоянием. При
необходимости заменить значение поля с кодированием типа, в него
подставляется другой объект-состояние.
:::
:::::

::::: dl
::: dt
[Replace Subclass with Fields](../../replace-subclass-with-fields.html)
:::

::: dd
**Проблема:** У вас есть подклассы, которые отличаются только методами,
возвращающими данные-константы.

**Решение:** Замените методы полями в родительском классе и удалите
подклассы.
:::
