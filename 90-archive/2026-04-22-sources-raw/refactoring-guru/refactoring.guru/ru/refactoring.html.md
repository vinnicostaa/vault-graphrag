# Рефакторинг {#рефакторинг .ref1-h .ref-abs .ref-c .ref-h1}

::: mob-image
[![Вступление в
Рефакторинг](../images/refactoring/content/index/refactoring-i166e6.png?id=4ba85db5ddb3da0cd829bc7c0d2bf407){width="256"
height="192"
srcset="/images/refactoring/content/index/refactoring-i1-2x.png?id=dc9209e9ccee993f51c33697c716aa49 2x,/images/refactoring/content/index/refactoring-i1-3x.png 3x"}](refactoring/what-is-refactoring.html "Вступление в Рефакторинг")
:::

Рефакторинг --- это контролируемый процесс улучшения кода, без написания
новой функциональности. Результат рефакторинга --- это чистый код и
простой дизайн.

.btn-primary}
:::

::: mob-image
[![Грязный

побочным продуктом частых изменений в процессе разработки.

###  Чистый код {#чистый-код .ref3-h .ref-abs .ref-r .ref-h3}

::: mob-image
[![Чистый
код](../images/refactoring/content/index/refactoring-i3179f.png?id=e393546f4f2d0046fdbf98b75c6c35ea){width="256"
height="192" loading="lazy"
srcset="/images/refactoring/content/index/refactoring-i3-2x.png?id=ca4e076f5522aa5d861350a7a741d1d4 2x,/images/refactoring/content/index/refactoring-i3-3x.png 3x"}](refactoring/what-is-refactoring.html "Чистый код")
:::

Чистый код --- это код, который просто читать, понимать и поддерживать.
Чистый код улучшает предсказуемость разработки и повышает качество
продукта.

###  Запахи кода {#запахи-кода .ref5-h .ref-abs .ref-l .ref-h3}

::: mob-image
[![Запахи
кода](../images/refactoring/content/index/refactoring-i5a1cd.png?id=acb5ca2349f078fa48ea57b5ad1ee0f5){width="256"
height="192" loading="lazy"

Запахи кода --- это индикаторы проблем, на которые нужно обращать
внимание при рефакторинге. Часто их легко найти и исправить, но иногда
они предвещают о глубинных проблемах с кодом.

::: mob-image
[![Техники
рефакторинга](../images/refactoring/content/index/refactoring-i646d6.png?id=7f2e9d6b728193f982cfc4d2dbd384d0){width="256"

Техники рефакторинга описывают конкретные методы борьбы с грязным кодом.
