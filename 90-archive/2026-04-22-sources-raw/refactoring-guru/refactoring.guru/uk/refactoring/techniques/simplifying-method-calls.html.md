# Спрощення викликів методів {#спрощення-викликів-методів .title}

Ці рефакторинги роблять виклики методів простіше і ясніше для розуміння.
Це, в свою чергу, спрощує інтерфейси взаємодії між класами.

::::::::::::::::::::::::::::::::::::::::::::: {.relations .link-list}
::::: dl
::: dt
[Rename Method](../../rename-method.html)
:::

::: dd
**Проблема:** Назва методу не розкриває суть того, що він робить.

**Рішення:** Змініть назву методу.
:::
:::::

::::: dl
::: dt
[Add Parameter](../../add-parameter.html)
:::

::: dd
**Проблема:** Методу не вистачає даних для здійснення якихось дій.

**Рішення:** Створіть новий параметр, щоби передати ці дані.
:::
:::::

::::: dl
::: dt
[Remove Parameter](../../remove-parameter.html)
:::

::: dd
**Проблема:** Параметр не використовується в тілі методу.

**Рішення:** Видаліть невживаний параметр.
:::
:::::

::::: dl
::: dt
[Separate Query from Modifier](../../separate-query-from-modifier.html)
:::

::: dd
**Проблема:** У вас є метод, який повертає якесь значення, але при цьому
в процесі роботи він змінює щось всередині об'єкта.

**Рішення:** Розділіть метод на два різних методи. Один з них нехай
повертає значення, а другий модифікує об'єкт.
:::
:::::

::::: dl
::: dt
[Parameterize Method](../../parameterize-method.html)
:::

::: dd
**Проблема:** Декілька методів виконують схожі дії, які відрізняються
тільки якимись внутрішніми значеннями, числами або операціями.

**Рішення:** Об'єднайте усі ці методи в один з параметром, в який
передаватиметься значення, що відрізняється.
:::
:::::

::::: dl
::: dt
[Replace Parameter with Explicit
Methods](../../replace-parameter-with-explicit-methods.html)
:::

::: dd
**Проблема:** Метод було розбито на частини, кожна з яких виконується
залежно від значення якогось параметра.

**Рішення:** Витягніть окремі частини методу у власні методи і
викликайте їх замість оригінального методу.
:::
:::::

::::: dl
::: dt
[Preserve Whole Object](../../preserve-whole-object.html)
:::

::: dd
**Проблема:** Ви отримуєте декілька значень від об'єкта, а потім
передаєте їх в метод як параметри.

**Рішення:** Замість цього передавайте весь об'єкт.
:::
:::::

::::: dl
::: dt
[Replace Parameter with Method
Call](../../replace-parameter-with-method-call.html)
:::

::: dd
**Проблема:** Викликаємо метод і передаємо його результати в параметри
іншого методу. При цьому значення параметрів могли б бути отримані і
всередині викликаного методу.

**Рішення:** Замість передачі значення через аргументи, спробуйте
перемістити код отримання значення всередину самого методу.
:::
:::::

::::: dl
::: dt
[Introduce Parameter Object](../../introduce-parameter-object.html)
:::

::: dd
**Проблема:** У ваших методах зустрічається група параметрів, що
повторюється.

**Рішення:** Замініть ці параметри об'єктом.
:::
:::::

::::: dl
::: dt
[Remove Setting Method](../../remove-setting-method.html)
:::

::: dd
**Проблема:** Значення поля має бути встановлене тільки в момент
створення і більше ніколи не мінятися.

**Рішення:** Видаліть методи, що встановлюють значення цього поля.
:::
:::::

::::: dl
::: dt
[Hide Method](../../hide-method.html)
:::

::: dd
**Проблема:** Метод не використовується іншими класами або
використовується тільки всередині своєї ієрархії класів.

**Рішення:** Зробіть метод приватним або захищеним.
:::
:::::

::::: dl
::: dt
[Replace Constructor with Factory
Method](../../replace-constructor-with-factory-method.html)
:::

::: dd
**Проблема:** У вас є складний конструктор, що робить щось більше, ніж
просте встановлення значень для полів об'єкта.

**Рішення:** Створіть фабричний метод і замініть ним виклики
конструктора.
:::
:::::

::::: dl
::: dt
[Replace Error Code with
Exception](../../replace-error-code-with-exception.html)
:::

::: dd
**Проблема:** Метод повертає певне значення, яке сигналізуватиме про
помилку.

**Рішення:** Замість цього слід викидати виключення.
:::
:::::

::::: dl
::: dt
[Replace Exception with Test](../../replace-exception-with-test.html)
:::

::: dd
**Проблема:** Ви викидаєте виключення там, де можна було б обійтися
простою перевіркою умови.

**Рішення:** Замініть викидання виключення перевіркою цієї умови.
:::
