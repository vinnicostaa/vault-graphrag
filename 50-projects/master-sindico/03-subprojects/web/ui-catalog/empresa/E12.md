# E12 — Perfil Institucional

## Finalidade
Vitrine pública da empresa visualizada por síndicos quando consultam fornecedores na plataforma. Apresenta dados institucionais, logo, descrição, segmento, vídeos institucionais (gerenciados pela agência via [[../marketing/MK4]]), portfolio e média de avaliações ([[E11]]).

## Fonte canônica
- `vault/90-archive/master-sindico-pdfs-cliente-originais/JORNADA DA EMPRESA-1.md` (TELA E11 — Perfil Institucional)
- Referência cruzada: agência publica vídeos via [[../marketing/MK4]] (delegação)

## Persona e ABAC
- **Persona primária visualizadora**: Síndico (consulta) + Empresa (edita o próprio)
- **Plan tier mínimo**: `plus`
- **ABAC action**: `enterprise.profile.read` (público) + `enterprise.profile.update` (próprio admin)
- **Restrições**: edição apenas Administrador; vídeos institucionais gerenciados pela agência delegada via [[../marketing/MK3]].

## Fluxo da tela
1. Empresa em E1 → clica card "Perfil institucional".
2. Sistema carrega dados completos da empresa.
3. Empresa Administrador vê toolbar [Editar perfil].
4. Em modo edição: campos editáveis (lista abaixo); upload de logo; lock 90d em vídeos institucionais (D-066).
5. Vídeos institucionais aparecem em galeria — gerenciados pela agência ([[../marketing/MK4]] / [[../marketing/MK5]]).
6. Portfolio (cases de execução) é gerado automaticamente das execuções publicadas em [[E9]] (pode ocultar).
7. Empresa salva → perfil atualizado e visível aos síndicos via consulta.

## Componentes
- `ProfileHeader` (logo + nome fantasia + segmento + média avaliação)
- `EditableForm` (campos abaixo)
- `LogoUploader` (PNG/JPG max 2MB)
- `VideoGallery` (read-only — populada pela agência via MK4)
- `PortfolioGrid` (execuções publicadas — ocultar individualmente)
- `RatingDisplay` (média 5 critérios — fonte [[E11]])
- `SaveButton`

## Campos editáveis
- Razão social
- Nome fantasia
- CNPJ (somente leitura após onboarding)
- Cidade / Estado
- Telefone
- Email institucional
- Site
- Logo
- Descrição institucional (texto + voz, R2)
- Segmento de atuação (1 das 31 categorias canônicas — fonte F8)
- Subcategorias do serviço (múltipla — ~180 subcategorias — fonte F8)
- Áreas de cobertura (cidades/bairros)

## Estados
- **View mode**: Síndico ou Empresa não-admin vê read-only
- **Edit mode**: Administrador edita
- **Loading**: skeleton
- **Saving**: spinner no botão Salvar; campos disabled
- **Error**: toast retry; conteúdo preservado
- **Validation-error**: bordas vermelhas em campos inválidos (CNPJ, email, URL)

## Integração com backend
| Ação UI | Endpoint | Método | Payload |
|---|---|---|---|
| Buscar perfil | `/api/v1/enterprise/profile` | GET | Company completo |
| Atualizar perfil | `/api/v1/enterprise/profile` | PUT | partial fields |
| Upload logo | `/api/v1/media/upload?type=logo` | POST | multipart |
| Listar vídeos institucionais | `/api/v1/content/videos?company_id=:id` | GET | Video[] |

Use cases `enterprise.UpdateProfile` + `content.ListCompanyVideos`.

## Regras de negócio críticas
- **Vídeos**: empresa NÃO publica vídeos diretamente — apenas a agência delegada via [[../marketing/MK4]]
- **Visibilidade vídeos**: moradores NÃO acessam vídeos institucionais normalmente — só durante votação fornecedor em assembleia (D-064)
- **Lock 90 dias** em vídeos institucionais (D-066 — confirmar para vídeos de empresa)
- **Avaliação média**: calculada com base nas avaliações públicas ([[E11]])
- **31 categorias + subcategorias canônicas** (fonte F8 — `enterprise-categories-subcategories-enum.schema.ts`)
- **CNPJ imutável** após onboarding (audit trail)
