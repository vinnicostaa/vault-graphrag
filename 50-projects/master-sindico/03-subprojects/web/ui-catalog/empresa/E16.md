# E16 — Formulário Marketing Link

## Finalidade
Empresa preenche formulário direcionado a uma agência de marketing específica (acessada via perfil da agência) registrando **intenção de contato** sobre serviços de marketing institucional. **NÃO cria vínculo automático** — apenas notifica a agência via [[../marketing/MK8]]. Vínculo formal só via [[E14]].

## Fonte canônica
- `vault/90-archive/master-sindico-pdfs-cliente-originais/JORNADA DA EMPRESA.md` (TELA E16 — Formulário Marketing Link + REGRAS OPERACIONAIS)
- `vault/90-archive/from-development-content-2026-04-23/03-Modulos/Connect-Me-4-direcoes.md` (Marketing Link — clarificação sobre não-vínculo)

## Persona e ABAC
- **Persona primária**: Empresa (Administrador — operador técnico bloqueado)
- **Plan tier mínimo**: `plus`
- **ABAC action**: `commercial.marketing_link.create`
- **Restrições**: deve ser acessada a partir do perfil de uma agência específica (não há lista global de agências para spam).

## Fluxo da tela
1. Empresa navega no perfil de uma agência (vitrine pública das agências — referência cruzada).
2. Empresa clica botão [Marketing Link].
3. Sistema abre formulário com campos da empresa pré-preenchidos quando possível.
4. Empresa preenche os campos (nome empresa, responsável, telefone, email, tipo de apoio, prioridade, descrição).
5. Empresa clica [Enviar solicitação].
6. Sistema cria registro `marketing_link_requests` + dispara notificação para a agência (aparece em [[../marketing/MK8]]).
7. Agência recebe: nome empresa, responsável, telefone, email, descrição da necessidade.
8. Empresa pode acompanhar status (sent / viewed / atendido) — mas não há revelação cruzada como Connect Me.

## Componentes
- `MarketingLinkForm`
- `SupportTypeSelect` (6 opções)
- `PrioritySelect` (3 opções)
- `TextareaDescription`
- `ButtonGroup` (Enviar / Cancelar)
- `StatusTracker` (após envio, mostra status)

## Campos do formulário (fonte JORNADA DA EMPRESA.md)

### Empresa (pré-preenchidos)
- Nome da empresa
- Nome do responsável
- Telefone
- Email

### Necessidade de marketing — Tipo de apoio desejado (6 opções)
1. Produção de vídeos institucionais
2. Gestão de conteúdos técnicos
3. Estratégia de posicionamento no mercado condominial
4. Gestão de redes sociais
5. Consultoria de marketing para empresas de serviço
6. Outro (campo aberto)

### Prioridade (3)
- Imediato
- Nos próximos 30 dias
- Sem prazo definido

### Descrição da necessidade
- Campo aberto (placeholder: "Descreva brevemente o tipo de apoio que sua empresa procura.")

## Estados
- **Loading**: skeleton form
- **Validation-error**: bordas vermelhas
- **Sent**: toast "Solicitação enviada à agência" + redireciona para tela de acompanhamento (status `sent`)
- **Error**: toast retry; conteúdo preservado
- **Already-sent recently**: aviso "Você já enviou solicitação a esta agência em DD/MM/AAAA. Aguarde retorno antes de enviar nova."

## Integração com backend
| Ação UI | Endpoint | Método | Payload |
|---|---|---|---|
| Enviar Marketing Link | `/api/v1/commercial/marketing-link` | POST | { agency_id, support_type, priority, description, contact_data } |
| Status do link | `/api/v1/commercial/marketing-link/:id` | GET | MarketingLinkRequest |

Use case `commercial.CreateMarketingLink` + persistência em tabela própria `marketing_link_requests` (separada de `connect_me_requests`).

## Regras de negócio críticas
- **NÃO cria vínculo automático** entre empresa e agência (regra dura — fonte F4)
- **Vínculo só por convite formal** via [[E14]] "Gestão de Agência de Marketing"
- **Status do Marketing Link** (6): sent / viewed / proposal_sent / atendido / accepted / rejected
- **Não tem quota** (não é volume sensitive como Connect Me)
- **Não é unidirecional puro** — é formulário de intenção; LGPD log opcional (não há revelação cruzada de dados sensíveis)
- **Anti-spam**: empresa não pode enviar > N Marketing Links para a mesma agência em janela de tempo (regra a confirmar)
