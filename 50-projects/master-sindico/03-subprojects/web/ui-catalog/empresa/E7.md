# E7 — Negócios Fechados

## Finalidade
Lista de todos os negócios fechados (contratos formalizados) entre a empresa e condomínios, com estados do ciclo de vida (aguardando confirmação dupla → confirmado → em execução → concluído → garantia ativa → garantia encerrada). Centro operacional pós-comercial.

## Fonte canônica
- `vault/90-archive/from-development-content-2026-04-23/contents/####################TELA DO MORADOR#####(1).md` (bloco "🔷 E7 — NEGÓCIOS FECHADOS")
- `vault/90-archive/from-development-content-2026-04-23/contents/####################TELA DO MORADOR#####(1).md` (TELA 31-34 negócio fechado pelo síndico)

## Persona e ABAC
- **Persona primária**: Empresa (Administrador)
- **Plan tier mínimo**: `plus`
- **ABAC action**: `commercial.closed_deal.list_own`
- **Restrições**: lista filtrada por `company_id`.

## Fluxo da tela
1. Empresa em E1 → clica card "Negócios fechados" (variante de Status de envios).
2. Sistema lista deals com filtros padrão (ativos primeiro).
3. Cada linha: Condomínio, Serviço, Valor final, Condições de pagamento, Prazo início, Prazo conclusão, Garantia, Status execução.
4. Empresa clica [Abrir] → detalhe do negócio (com termo de execução assinado, comunicados, registros).
5. A partir do detalhe pode: registrar nova execução (atalho [[E9]]), criar comunicado pós-fechamento (vai validação síndico), criar adendo formal (após confirmação dupla).

## Componentes
- `DealTable` (com badge status colorido)
- `FilterSidebar` (status execução, garantia, condomínio, período)
- `BadgeStatusDeal`
- `WarrantyExpiryAlert` (badge laranja se < 30 dias)
- `EmptyState`

## Estados do negócio (fonte ####TELA####(1).ini E7)
1. Aguardando confirmação dupla (empresa + síndico precisam assinar — ver [[E8]])
2. Confirmado (ambos assinaram, imutável)
3. Em execução (após confirmação)
4. Concluído (todos os registros validados pelo síndico)
5. Garantia ativa (período de garantia ainda válido)
6. Garantia encerrada (após prazo)

## Estados UI
- **Loading**: skeleton table
- **Empty**: "Sua empresa ainda não fechou negócios. Comece em [[E2]]."
- **Error**: toast retry
- **Pending action**: badge amarelo "Aguardando sua assinatura" se empresa não assinou termo

## Integração com backend
| Ação UI | Endpoint | Método | Retorno |
|---|---|---|---|
| Listar deals | `/api/v1/commercial/closed-deals/own` | GET | ClosedDeal[] |
| Filtrar | query string | GET | filtrado |
| Detalhe | `/api/v1/commercial/closed-deals/:id` | GET | ClosedDeal completo |

Use cases `commercial.ListOwnDeals`, `commercial.GetDeal`.

## Regras de negócio críticas
- **Confirmação dupla**: empresa assina [[E8]] + síndico assina (S33-S34) → estado vira "Confirmado" e fica imutável
- **R5 imutabilidade**: alteração só via Adendo Formal após confirmação
- **Alertas garantia**: notificações 30/15/7 dias antes do vencimento (componente `WarrantyExpiryAlert`)
- **Negócios fechados** alimentam dashboard E10 e influenciam reputação/avaliação (E11)
