# E14 — Gestão de Agência (delegação)

## Finalidade
Empresa convida e gerencia agência de marketing autorizada a administrar conteúdos institucionais (vídeos) em seu nome dentro da plataforma. **Esta é a tela que cria o vínculo formal** empresa↔agência (diferente do Marketing Link [[E16]] que apenas registra intenção).

## Fonte canônica
- `vault/90-archive/master-sindico-pdfs-cliente-originais/JORNADA DA EMPRESA-1.md` (TELAS E13/E14/E15 — Gestão de Agência + Convidar + Gerenciar Acessos)
- `vault/50-projects/master-sindico/00-product/sub-produtos/10-marketing-agencias.md` (D-059/D-060 sobre delegação)

## Persona e ABAC
- **Persona primária**: Empresa (apenas Administrador)
- **Plan tier mínimo**: `plus` (Trial não pode convidar agência — fonte personas-and-plans.md)
- **ABAC action**: `enterprise.agency.manage`
- **Restrições**: 1 empresa pode ter N agências mas geralmente 1 ativa por vez (regra a confirmar).

## Fluxo da tela
1. Empresa Administrador em E1 → clica card "Gestão de agência de marketing".
2. Sistema lista agências (status: Convite enviado / Ativa / Encerrada).
3. Administrador clica [Convidar agência] → abre formulário (nome, responsável, email, telefone) + checkbox de autorização.
4. Sistema envia convite via token (Zitadel) ao email da agência.
5. Agência aceita convite → loga ou cria conta → aparece em [[../marketing/MK2]] empresas atendidas.
6. Vínculo ativo: agência ganha grants ABAC scoped (apenas vídeos da empresa).
7. Administrador pode [Gerenciar acessos] → revogar acesso a qualquer momento (status vira Encerrada).

## Componentes
- `AgencyTable` (linhas com status badge)
- `InviteAgencyModal` (formulário + checkbox autorização)
- `ManageAccessDrawer` (lista de permissões da agência)
- `RevokeAccessDialog` (double-confirm)
- `StatusBadge` (Convite enviado / Ativa / Encerrada)

## Campos do convite (E14)
- Nome da agência
- Responsável
- Email (obrigatório — chave de identidade)
- Telefone
- Checkbox: "Autorizo esta agência a administrar os conteúdos institucionais da minha empresa na plataforma Master Síndico" (obrigatório)

## Permissões da agência (gates ABAC — fonte 10-marketing-agencias.md §6)
- ✅ Publicar vídeos institucionais ([[../marketing/MK4]])
- ✅ Gerenciar biblioteca de vídeos ([[../marketing/MK5]])
- ✅ Visualizar dashboard da empresa ([[../marketing/MK6]])
- ❌ Acessar Connect Me / Timeline / dados síndico / dados morador / dashboard governança

## Estados
- **Loading**: skeleton table
- **Empty**: "Sua empresa ainda não tem agência delegada. Convide uma agência para gerenciar seus vídeos institucionais."
- **Error**: toast retry
- **Convite enviado**: aguardando aceite agência — badge laranja
- **Ativa**: agência operando — badge verde
- **Encerrada**: vínculo revogado — badge cinza (R3 — histórico preservado)

## Integração com backend
| Ação UI | Endpoint | Método | Payload |
|---|---|---|---|
| Listar agências | `/api/v1/enterprise/agencies` | GET | Agency[] |
| Convidar agência | `/api/v1/enterprise/agencies/invite` | POST | { name, responsible, email, phone, authorize: true } |
| Revogar acesso | `/api/v1/enterprise/agencies/:id/revoke` | POST | - |
| Listar permissões | `/api/v1/enterprise/agencies/:id/permissions` | GET | Permission[] |

Use cases `enterprise.InviteAgency` + `enterprise.RevokeAgency` + integração Zitadel.

## Regras de negócio críticas
- **D-059/D-060**: Agência é sub-produto interno próprio — login Zitadel próprio + grant ABAC scoped (não impersonation)
- **Marketing Link [[E16]] ≠ vínculo**: apenas registra intenção; vínculo formal só por esta tela
- **Trial bloqueado**: Empresa Trial NÃO pode convidar agência (gate comercial)
- **Audit trail obrigatório**: convite, aceite, revogação geram logs imutáveis
- **Revogação imediata**: ao encerrar, agência perde grants ABAC instantaneamente; vídeos publicados permanecem (apenas gestão futura bloqueada)
- **1 agência ↔ N empresas** (agência atende várias) e **1 empresa ↔ N agências** (mas geralmente 1 ativa)
