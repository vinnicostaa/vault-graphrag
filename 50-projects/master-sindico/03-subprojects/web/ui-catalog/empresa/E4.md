# E4 — Enviar Proposta

## Finalidade
Após demonstrar interesse em uma oportunidade Connect Me (E2 → "Tenho interesse"), a empresa visualiza o contato do síndico (revelação LGPD-logada) e formaliza uma proposta técnica e comercial. A proposta entra na trilha de validação do síndico (S27).

## Persona e ABAC
- **Persona primária**: Empresa (Administrador — operador técnico não envia proposta)
- **Plan tier mínimo**: `plus`
- **ABAC action**: `commercial.proposal.create` + `commercial.connect_me.express_interest`
- **Restrições**: Connect Me precisa estar em `pending` no momento do clique "Tenho interesse".

## Fluxo da tela
1. Empresa em E2 clica [Tenho interesse].
2. Sistema atualiza `connection_status = interested` + cria registro LGPD (timestamp, IP, terms_version, finalidade).
3. Sistema revela: nome do síndico, telefone, email, descrição completa da demanda.
4. Empresa preenche formulário de proposta (campos abaixo).
5. Empresa pode [Salvar rascunho] ou [Enviar proposta].
6. Ao enviar: status muda para `proposal_sent`; síndico recebe notificação para validar (vai a S27).
7. Botões auxiliares: [Entrar em contato] (atalho mailto/tel), [Marcar como em negociação], [Encerrar oportunidade].

## Componentes
- `ContactRevealCard` (nome síndico + telefone + email + LGPD badge "Dados liberados em DD/MM/AAAA HH:MM")
- `ProposalForm` (campos abaixo)
- `FileUpload` (PDF da proposta)
- `ButtonGroup` (Salvar rascunho / Enviar proposta / Cancelar)
- `StatusBanner` ("Proposta aguardando validação do síndico")

## Campos obrigatórios da proposta (fonte ####TELA####(1).ini)
- Descrição técnica detalhada
- Escopo delimitado
- Valor total
- Condições de pagamento
- Garantia oferecida
- Upload da proposta (PDF)

## Estados
- **Loading**: skeleton form + spinner reveal contato
- **Empty**: N/A
- **Error**: toast "Não foi possível enviar a proposta" + retry; rascunho preservado
- **Draft saved**: badge "Rascunho salvo" + autosave a cada 30s
- **Sent**: redireciona para [[E5]] (Minhas propostas)
- **Expired**: oportunidade vencida (48h sem reveal) → estado read-only com mensagem "Esta oportunidade expirou"

## Integração com backend
| Ação UI | Endpoint | Método | Payload |
|---|---|---|---|
| Express interest (reveal) | `/api/v1/commercial/connect-me/:id/interested` | POST | - |
| Criar proposta | `/api/v1/commercial/proposals` | POST | { connect_me_id, ...fields } |
| Salvar rascunho | `/api/v1/commercial/proposals/:id/draft` | PUT | partial fields |
| Enviar para validação | `/api/v1/commercial/proposals/:id/submit` | POST | - |

Use cases: `commercial.RespondConnectMe`, `commercial.CreateProposal`, `commercial.SubmitProposal`.

## Regras de negócio críticas
- **Reveal único**: clicar "Tenho interesse" é ação irreversível (gera log LGPD imutável)
- **Prazo síndico-definido**: síndico define prazo para envio de proposta (fonte ####TELA####(1).ini)
- **Validação obrigatória**: proposta só aparece aos moradores APÓS validação do síndico (S27)
- **Versionamento**: termo LGPD vigente no momento do reveal é registrado (auditoria)
