# E15 — Compliance Empresarial

## Finalidade
Painel de compliance da empresa: declarações anuais (fiscal, técnica, trabalhista), histórico LGPD, controle de garantias, módulo de adendos. **Sem declaração anual atualizada → bloqueio de novas propostas** (gate operacional). Equivalente empresarial do Compliance do Síndico (S28-S31, C1-C11).

## Fonte canônica
- `vault/90-archive/from-development-content-2026-04-23/contents/####################TELA DO MORADOR#####(1).md` (bloco "🔷 E14 — COMPLIANCE EMPRESARIAL")

## Persona e ABAC
- **Persona primária**: Empresa (Administrador — Operador Técnico em modo leitura)
- **Plan tier mínimo**: `plus`
- **ABAC action**: `enterprise.compliance.manage`
- **Restrições**: assinatura de declarações apenas Administrador (com MFA obrigatório M2+).

## Fluxo da tela
1. Empresa Administrador em E1 → clica card "Compliance".
2. Sistema mostra status geral: Em dia / Parcial / Pendente.
3. Painel lista pendências: declaração fiscal, técnica, trabalhista (cada uma com data de vencimento).
4. Administrador clica em uma declaração → abre tela de leitura + checkbox + assinatura.
5. Após assinar: estado vira "Em dia"; gera publicação automática na Linha do Tempo dos condomínios atendidos (visibilidade interna gestão).
6. Histórico LGPD: log de todas as revelações de dados (Connect Me, etc.).
7. Controle de garantias: lista de garantias vencendo nos próximos 30/60/90 dias.
8. Adendos: lista de adendos formais criados em deals confirmados.

## Componentes
- `ComplianceStatusBanner` (Em dia / Parcial / Pendente — colorido)
- `DeclarationCard` (3 cards: fiscal, técnica, trabalhista — cada um com status + ação)
- `LGPDLogTable` (histórico revelações)
- `WarrantyAlertsList`
- `AddendumList`
- `SignDeclarationDialog`

## Declarações anuais (fonte ####TELA####(1) E14)
1. **Declaração anual de regularidade fiscal**
2. **Declaração anual de regularidade técnica**
3. **Declaração anual trabalhista**

## Estados
- **Em dia**: badge verde — todas as 3 declarações vigentes
- **Parcial**: badge amarelo — pelo menos 1 pendente mas dentro do prazo
- **Pendente**: badge vermelho — declaração vencida → **bloqueia novas propostas** (gate)
- **Loading**: skeleton
- **Error**: toast retry

## Integração com backend
| Ação UI | Endpoint | Método | Payload |
|---|---|---|---|
| Status compliance | `/api/v1/enterprise/compliance/status` | GET | { fiscal, tecnica, trabalhista, lgpd_count, warranty_alerts } |
| Assinar declaração | `/api/v1/enterprise/compliance/declarations/:type/sign` | POST | { accepted: true } |
| Histórico LGPD | `/api/v1/enterprise/compliance/lgpd-logs` | GET | LGPDLog[] |
| Garantias vencendo | `/api/v1/enterprise/compliance/warranties` | GET | Warranty[] |
| Adendos | `/api/v1/enterprise/compliance/addendums` | GET | Addendum[] |

Use cases `enterprise.SignAnnualDeclaration` + `enterprise.GetComplianceDashboard`.

## Regras de negócio críticas
- **Bloqueio operacional**: declaração anual vencida → gate ABAC bloqueia [[E4]] (nova proposta) e [[E2]] (aceitar oportunidades)
- **Renovação anual**: sistema dispara notificação 30 dias antes do vencimento
- **Imutabilidade**: declaração assinada é registro imutável (R5 + LGPD log)
- **Auto-publicação**: cada assinatura gera item institucional na Linha do Tempo dos condomínios atendidos (similar a S29-S30)
- **Adendo formal**: único mecanismo de alteração após confirmação dupla em [[E8]]
