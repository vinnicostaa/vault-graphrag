# BT11 — Confirmação

## Finalidade
Tela final do fluxo de cadastro. Mensagem de sucesso + resumo do que acontece a seguir (perfil visível a empresas, podem visualizar vídeo + respostas para avaliar compatibilidade).

## Fonte canônica

## Mensagem canônica (literal)
> "Cadastro concluído com sucesso!
>
> Seu perfil profissional agora faz parte da Master Síndico. As empresas poderão visualizar suas informações, respostas e vídeo para avaliar compatibilidade com o ambiente de trabalho.
>
> Você pode atualizar seu perfil sempre que quiser. Aqui, comportamento e alinhamento importam."

## Persona e ABAC
- **Persona**: Morador Pagante (`resident`)
- **Plan tier**: `base` pagante
- **ABAC action**: `content.talent.curriculum.submit.self`

## Conteúdo
- Mensagem de sucesso celebrativa
- Resumo do currículo (preview compacto)
- Status: `submitted` (aguardando moderação admin)
- Estimativa de tempo de aprovação (sugerido 1-2 dias úteis — gap)
- Botões: **Voltar para o início** / **Editar perfil**

## Componentes
- `SuccessHero` (animação + mensagem)
- `CurriculumPreview` (cards das seções)
- `StatusBanner` (Aguardando aprovação)
- `EditButton` (volta para [[BT01]])

## Estados
- **Submitting**: spinner (durante POST final).
- **Success**: tela completa visível.
- **Submission failed**: toast com retry; preserva conteúdo.
- **Already published**: status verde + link para "Editar perfil".
- **Rejected previously**: banner com motivo + CTA "Editar e re-submeter".
- **Error**: toast retry.

## Fluxo backend
1. POST final transiciona `Curriculum.status = draft → submitted`.
2. Validações server-side (LGPD aceito, vídeo ≤ 90s, ≥ 1 área, ≥ 1 modalidade, formação preenchida).
3. Admin review (síncrono manual M3; automação M4+).
4. Aprovação → `submitted → published` → evento `content.talent.curriculum-published` → indexa em busca.
5. Rejeição → `submitted → draft` com motivo; morador reabre/corrige.

## Integração com backend
| Ação | Endpoint (inferido) | Método |
|---|---|---|
| Submeter currículo | `/api/v1/talents/curriculum/me/submit` | POST |
| Status atual | `/api/v1/talents/curriculum/me` | GET |

## Regras de negócio críticas
- Validações server-side antes de aceitar `submitted`.
- Estados: `draft → submitted → published → archived` (state machine §12 do sub-produto).
- Empresa NÃO acessa esta tela (vê resultados de busca em outras telas — fora deste catálogo).
- Eventos: `content.talent.curriculum-created/updated/published`.
- Após `published`, currículo aparece nas buscas das Empresas Plus/Pro.
- Toda busca empresa é logada em `curriculum_search_logs` (LGPD).
