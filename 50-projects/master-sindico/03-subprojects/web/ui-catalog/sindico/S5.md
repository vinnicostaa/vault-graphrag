# S5 — Identidade do Condomínio (ID/QR)

## Finalidade
Etapa 3 de 3 do cadastro: o sistema gera automaticamente o identificador alfanumérico oficial e o QR Code do condomínio. É a credencial usada por moradores para se vincularem (M3 — busca por ID).

## Persona e ABAC
- **Persona primária**: Síndico (`syndic`, role_type: `principal`)
- **Plan tier mínimo**: `trial`
- **ABAC action**: `condominium.finalize` + `invite.share`
- **Restrições**:
  - Concluir requer S3 + S4 completos (rascunho consistente).
  - ID gerado é **único globalmente** (collision check) e formato alfanumérico seguro (entropia mínima — assumir 12 chars base32 = ~60 bits).

## Fluxo da tela
1. Sistema persiste o condomínio (sai de `draft` para `active`) e gera: `condo_id` alfanumérico + QR Code apontando para deep link `mastersindico://join?id={ID}` (inferido) + URL HTTPS espelho.
2. Renderiza card grande com ID, QR e mensagem fixa: "Este é o identificador oficial do seu condomínio na Master Síndico. Use para convidar moradores com segurança."
3. Botões: "Copiar ID" · "Compartilhar convite" (modal com mensagem padrão editável) · "Concluir e acessar Central" → S6.
4. Mensagem padrão para compartilhamento (editável): "Acompanhe tudo o que está sendo realizado no nosso condomínio com transparência e organização. Baixe o app Master Síndico e use o ID: [ID]. Sua participação fortalece a gestão."

## Componentes
- `IDCard` (display destacado do alfanumérico) — link `[[../../design-system]]`
- `QRCodeView` (canvas com download PNG)
- `CopyButton` (clipboard API)
- `ShareModal` (mensagem editável + canais: WhatsApp, e-mail, link copy)
- `PrimaryButton` ("Concluir e acessar Central")

## Estados
- **Loading**: skeleton enquanto backend gera ID/QR.
- **Empty**: não aplicável (sempre tem dado pós-geração).
- **Error**: se geração falhar, exibir tela de erro com botão "Tentar novamente"; rascunho não é perdido.
- **Success**: redireciona automaticamente para S6 ao clicar "Concluir".

## Regras de negócio críticas
- ID é "sequência difícil de decifrar" (cliente .ini TELA 5) — usar gerador criptográfico, não auto-increment.
- QR Code aponta para o mesmo `condo_id` — moradores escaneiam e caem em M3 (Buscar Condomínio por ID).
- Mensagem padrão de compartilhamento é **editável** mas com modelo padrão (cliente .ini TELA 5).
- Após finalização, condomínio entra na contagem de hard limit (15 por síndico).
- Auto-publica TimelineEntry inicial: tipo `administrativo → conformidade_e_governanca` (inferido — alinhado a C2 padrão).
