# S21 — Registros de Execução (hub)

## Finalidade
Hub do síndico para registros de execução de serviços (vindos de empresas verificadas ou via link externo). Estes registros entram como "Aguardando validação" (R4) e precisam ser aprovados antes de irem para a Linha do Tempo pública.

## Fonte canônica
- `vault/90-archive/from-development-content-2026-04-23/contents/####################TELA DO MORADOR#####(1).md` (Bloco 8, TELA 16 — Serviços Hub; TELA 19 — Validação)
- `vault/90-archive/from-development-content-2026-04-23/02-Jornadas/Inventario-Telas.md` (S21 Validações)

## Persona e ABAC
- **Persona primária**: Síndico (`syndic`)
- **Plan tier mínimo**: `base`
- **ABAC action**: `service_record.list_pending`
- **Restrições**:
  - RequireTenantMatch.
  - Empresa síndica vinculada com permissão pode validar (Regra 6 PDF).

## Fluxo da tela
1. Header "Registros de Execução" + breadcrumb.
2. Tabs/cards:
   - **Pendentes de validação** (badge com contador)
   - **Validados**
   - **Rejeitados**
3. Filtros: empresa (autocomplete), categoria de serviço, período.
4. Lista (cronológica desc) com colunas: empresa (badge verificada/não verificada), categoria, data execução, status.
5. Click → S22 (detalhe + validação).
6. Botões: "Solicitar registro" (gera link para empresa externa ou seleciona cadastrada).

## Componentes
- `TabBar` (Pendentes | Validados | Rejeitados) — link `[[../../design-system]]`
- `FilterBar` (empresa, categoria, período)
- `ServiceRecordCard` (com badge empresa verificada/não)
- `EmptyState`
- `PrimaryButton` ("Solicitar registro")

## Estados
- **Loading**: skeleton.
- **Empty**: "Nenhum registro pendente" + CTA "Solicitar primeiro registro".
- **Error**: toast.
- **Success**: lista carregada.

## Integração com backend
| Ação | Endpoint | Método | Payload | Retorno |
|---|---|---|---|---|
| Listar registros | `/api/v1/condominiums/{id}/service-records?status=pending|validated|rejected` (inferido) | GET | — | `[{ id, empresa, categoria, data_execucao, status, verificada }]` |
| Solicitar registro (link externo) | `/api/v1/condominiums/{id}/service-records/external-link` (inferido) | POST | `{ prazo_inicio, prazo_termino }` | `{ link_url, expires_at }` |

## Regras de negócio críticas
- Origem externa sempre pendente (R4).
- Empresa não verificada exibe badge cinza.
- Síndico (ou empresa síndica vinculada com permissão) é o único validador.
- Após validação, registro vai para a LT como `registro_de_execucao`.
