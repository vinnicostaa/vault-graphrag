# S12 — Criar Atividade LT

## Finalidade
Publicar uma nova atividade da gestão na Linha do Tempo. Inclui mídia, classificação estruturada, visibilidade e modal anti-erro de confirmação de contexto.

## Fonte canônica
- `vault/90-archive/from-development-content-2026-04-23/contents/####################TELA DO MORADOR#####(1).md` (Bloco 7, TELA 15 — Novo Registro LT; R1 modal anti-erro; Bloco 5 listas mestre)
- `vault/90-archive/master-sindico-pdfs-cliente-originais/JORNADA DO SÍNDICO completa.md` (Regra 1)
- `vault/90-archive/from-development-content-2026-04-23/02-Jornadas/Inventario-Telas.md` (S12 LT)

## Persona e ABAC
- **Persona primária**: Síndico (`syndic`, principal ou usuário com toggle "Registrar atividade na Linha do Tempo")
- **Plan tier mínimo**: `base` (matriz funcional: criação requer pelo menos `base`; trial cria mas com soft-block)
- **ABAC action**: `timeline.create`
- **Restrições**:
  - RequireTenantMatch.
  - Visibilidade "Público aos moradores" requer toggle "Tornar registro público aos moradores".
  - Modal "Publicar para Condomínio X?" obrigatório antes de persistir (R1).

## Fluxo da tela
1. Form com seções (info, classificação, mídia, visibilidade).
2. Campos:
   - **Tipo de atividade** (lista 5.1)
   - **Título** (obrigatório)
   - **Descrição** (voz/texto — R2)
   - **Fotos** (upload múltiplo — opcional)
   - **Vídeos** (upload — opcional)
   - **Anexos** (PDF/doc — opcional)
   - **Áreas impactadas** (multi 5.2)
   - **Natureza** (lista 5.3)
   - **Importância** (lista 5.5)
   - **Risco** (categoria multi + nível único — 5.4)
   - **Status** (lista 5.8)
   - **Impacto esperado** (multi 5.6)
   - **Próxima ação** (multi 5.7)
   - **Vínculo a Ação PD** (opcional — autocomplete)
   - **Visibilidade** (toggle: Gestão | Público aos moradores)
3. Botão "Publicar" → abre modal "Publicar para Condomínio X?" → confirma → persiste.
4. Botão "Voltar".

## Componentes
- `Form` (multi-section) — link `[[../../design-system]]`
- `Select` / `MultiSelect` (listas mestre)
- `MediaUploader` (foto/vídeo/anexos com preview)
- `VoiceTextarea` (STT)
- `RiskMatrixField`
- `PDActionAutocomplete` (vincular ação PD)
- `VisibilityToggle` (Gestão | Público)
- `ConfirmContextModal` ("Publicar para Condomínio X?")

## Estados
- **Loading**: spinner durante upload + persistência.
- **Empty**: form vazio.
- **Error**: validação inline (título, tipo, natureza obrigatórios); toast em falha de upload.
- **Success**: redireciona para [[S11]] e exibe nova entry no topo.

## Integração com backend
| Ação | Endpoint | Método | Payload | Retorno |
|---|---|---|---|---|
| Upload mídia | `/api/v1/uploads` (inferido) | POST multipart | `file` | `{ url }` |
| Criar entry | `/api/v1/condominiums/{id}/timeline` (inferido) | POST | `{ tipo, titulo, descricao, midia[], areas[], natureza, importancia, risco_*, status, impacto[], proxima_acao[], pd_action_id?, visibilidade }` | `{ entry_id, created_at }` |

## Regras de negócio críticas
- Modal de confirmação anti-erro (R1) **obrigatório** antes de persistir — exibe Nome + Endereço resumido do condomínio.
- Após persistir, entry é **imutável** (R3). Edição de mídia ou texto exige adendo (S13 cria nova versão preservando histórico).
- Vínculo a Ação PD → reator backend atualiza status da PD (R2 PDF).
- Visibilidade "Público" expõe entry para morador em M6.
- Triggers de notificação interna automática (R9 .ini): "novo registro LT" para usuários relevantes.

## Gaps/ressalvas
- Cliente não define limite de upload de mídia — sugerido 10 fotos / 1 vídeo / 5 anexos por entry.
- Vinculação a Ação PD é "opcional", mas para o reator atualizar status PD (R2) deve ser obrigatória nesse cenário — gap de regra.
- STT não tem provider definido — sugerido whisper local ou Cloud STT.
