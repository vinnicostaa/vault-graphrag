# S13 — Editar Atividade LT (nova versão preservando histórico)

## Finalidade
Editar uma atividade já publicada na Linha do Tempo. Não sobrescreve; cria uma **nova versão** do TimelineEntry preservando o histórico imutável e marcando a versão anterior como `previous_entry_id`.

## Fonte canônica
- `vault/90-archive/from-development-content-2026-04-23/contents/####################TELA DO MORADOR#####(1).md` (Bloco 7 + Regra R3/R4 — preservação)
- `vault/90-archive/master-sindico-pdfs-cliente-originais/JORNADA DO SÍNDICO completa.md` (Regra 4 — Edição com preservação de histórico)
- `vault/90-archive/from-development-content-2026-04-23/02-Jornadas/Inventario-Telas.md` (S13 LT)

## Persona e ABAC
- **Persona primária**: Síndico (`syndic`, principal ou usuário com toggle "Registrar atividade na Linha do Tempo")
- **Plan tier mínimo**: `base`
- **ABAC action**: `timeline.amend` (edição = nova versão)
- **Restrições**:
  - RequireTenantMatch.
  - Empresa síndica vinculada com permissão pode editar publicações do síndico (Regra 6 PDF).
  - Após "negócio fechado" ou "votação encerrada", entries vinculadas só editáveis via adendo formal (R5 + C6).

## Fluxo da tela
1. Carrega entry alvo via `entry_id`.
2. Form pré-preenchido idêntico ao S12 com indicador "Você está editando — uma nova versão será criada".
3. Campo obrigatório adicional: "Justificativa da alteração" (texto, voz/texto).
4. Modal anti-erro antes de salvar: "Publicar nova versão para Condomínio X?".
5. Persiste como nova entry com `previous_entry_id = entry_anterior` + atualiza ponteiro `current_version_id`.
6. Botões: "Salvar nova versão" · "Voltar".

## Componentes
- `EditingBanner` ("Nova versão será criada — original preservado") — link `[[../../design-system]]`
- `Form` pré-populado (idem S12)
- `JustificationField` (obrigatório)
- `ConfirmContextModal` ("Publicar nova versão para Condomínio X?")
- `VersionPreview` (diff lado-a-lado opcional)

## Estados
- **Loading**: carregando entry original.
- **Empty**: não aplicável.
- **Error**: validação (justificativa obrigatória); falha de upload.
- **Success**: redireciona para [[S14]] (histórico) com a nova versão destacada.

## Integração com backend
| Ação | Endpoint | Método | Payload | Retorno |
|---|---|---|---|---|
| Buscar entry | `/api/v1/condominiums/{id}/timeline/{entry_id}` (inferido) | GET | — | entry completo |
| Criar nova versão | `/api/v1/condominiums/{id}/timeline/{entry_id}/versions` (inferido) | POST | `{ campos atualizados..., justificativa }` | `{ new_entry_id, previous_entry_id }` |

## Regras de negócio críticas
- **Nunca sobrescreve** — sempre INSERT (R3 .ini, R4 PDF).
- Justificativa **obrigatória** (compliance + audit C5).
- Listagem da LT exibe a versão corrente; histórico mostra todas (S14).
- Notificação interna automática (R9): "atividade X foi atualizada".
- Se entry está "bloqueada" (vinculada a negócio fechado/votação encerrada), botão Editar é substituído por "Criar adendo formal" (C6).
