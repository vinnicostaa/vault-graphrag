# S16 — Criar Evento (18 tipos canônicos + subcategorias)

## Finalidade
Publicar um novo evento no condomínio. Suporta **18 tipos canônicos** via [[../../../../STATE#d-115|D-115]] (Opção C) + subcategoria `equipment_target` para manutenção/inspeção técnica.

**Importante — NÃO faz aqui**:
- **Comunicados** (6 tipos que estavam no Bloco 9 do cliente): vão em [[S24]] Criar Comunicado com enum `announcement_type`
- **Enquete Consultiva**: vai em [[S11]] Criar Atividade LT tipo "poll" (aggregate [[../../../../01-domain/aggregates/Poll]])
- **Reservas de espaço comum** (churrasqueira, salão, piscina): novo sub-produto [[../../../../00-product/sub-produtos/12-reservas-espacos]] M3+
- **Assembleia**: criação real é em ASM-CFG (módulo assembleia live). S16 mostra Event **shadow** gerado automaticamente, mas não permite criar direto.

## Persona e ABAC

- **Persona primária**: Síndico (`syndic`, principal ou usuário com toggle "Criar evento")
- **Plan tier mínimo**: `base` (morador pago, síndico base, empresa base — todos podem criar se tiverem papel de síndico)
- **ABAC action**: `event.create`
- **Restrições**:
  - Permissão granular (matriz ABAC)
  - `RequireTenantMatch: true`
  - Modal de confirmação anti-erro antes de publicar (R1)
  - Tipo `assembleia_ordinaria` ou `assembleia_extraordinaria` **bloqueado no UI** (erro "use ASM-CFG") — se tentar pela API, `ErrBusiness`

## Fluxo da tela

1. Form em seções.
2. **Campos**:
   - **Tipo de evento** (Select 16 opções — assembleias removidas da lista, são shadow only):
     - Reunião de Conselho · Reunião Administrativa · Prestação de Contas · Audiência Jurídica · Apresentação de Projeto · Reunião com Fornecedor · Manutenção Programada · Inspeção Técnica · Obra Programada · Interrupção Programada de Serviços · Treinamento de Equipe · Simulado de Emergência · Campanha Institucional · Evento Comunitário · Atualização Normativa · Outros
   - **Equipamento alvo** (Select condicional — aparece só se tipo ∈ {Manutenção Programada, Inspeção Técnica}):
     - Elevador · Piscina · Jardim · Gerador · Hidráulico · Elétrico · Portão Automático · Sistema de Incêndio (AVCB) · CFTV · Climatização · Fachada · Telhado · Estrutura · Outros
   - **Título** (obrigatório, máx 120 chars)
   - **Descrição** (textarea, opcional, máx 2000 chars)
   - **Data início** (obrigatório, futura) + **Data fim** (opcional)
   - **Local** (opcional; recomendado obrigatório para tipos presenciais — registra como ressalva mas não bloqueia no MVP)
   - **Anexos** (opcional — fotos, PDFs de normativas/laudos/projetos)
   - **Visibilidade**: Gestão | Público (morador vê se Público)
   - **Notificar empresa contratada?** (checkbox — aparece para tipos operacionais: Manutenção/Inspeção/Obra/Interrupção — trigger email para empresa no ClosedDeal ativo desse escopo, se houver)
3. Botão "Publicar" → modal "Criar evento no Condomínio X?" → confirma → backend valida ABAC+invariantes → publica
4. Botão "Voltar" volta para [[S15]]

## Componentes

- `Form` (multi-section) + FormField (validation)
- `Select` (tipo de evento — 16 valores; assembleia_* filtradas fora)
- `Select` condicional (equipment_target — aparece se tipo ∈ {manutencao, inspecao})
- `DateTimeField` (scheduledAt futuro)
- `TextField` (location)
- `FileUpload` (anexos, múltiplos)
- `VisibilityToggle` (gestao | publico)
- `Checkbox` (notificar empresa)
- `ConfirmContextModal` (R1 anti-erro)
- Todos consumem tokens canônicos [[../../../../02-architecture/design-tokens]]

## Estados

- **Loading**: spinner ao publicar; após 3s aviso "Processando..."
- **Empty (form inicial)**: só tipo de evento selecionado destrava demais campos
- **Error**:
  - Validação: tipo, título, data início obrigatórios; data no futuro
  - Tipo manutenção/inspeção sem equipment: "Selecione o equipamento alvo"
  - Tentativa de criar `assembleia_ordinaria|extraordinaria`: "Eventos de assembleia são criados via Configurar Assembleia → [CTA redirect ASM-CFG]"
  - Backend 403: "Sem permissão para criar eventos"
- **Success**: toast "Evento publicado" + redireciona para [[S15]]

## Integração com backend

| Ação UI | Endpoint | Método | Payload | Retorno |
|---|---|---|---|---|
| Criar evento | `/api/v1/condominiums/{id}/events` | POST | `{ event_type, equipment_target?, title, description, scheduled_at, scheduled_until?, location?, attachments[], visibility, notify_contractor? }` | `201 { event_id }` |
| Pré-validar | `/api/v1/condominiums/{id}/events/validate` | POST | idem | `200 { valid: bool, warnings[], errors[] }` |

Backend endpoint a formalizar em [[../../../backend/requirements/institutional]] Sprint 10.

## Regras de negócio críticas

- **Enum fechado** (`event_type`): 18 valores canônicos em [[../../../../01-domain/aggregates/Event]]
- **Subcategoria condicional**: `equipment_target` obrigatório se tipo ∈ {manutencao_programada, inspecao_tecnica}, proibido nos demais (INV-EVT-003)
- **Assembleias são shadow Events** (INV-EVT-004): criação real em ASM-CFG; S16 **não oferece** opção de criar assembleia
- **`scheduledAt > NOW()`** (INV-EVT-001): não retroativo
- **Após publicar**: evento entra na timeline (auto-publicação via evento de domínio `EventScheduled`)
- **Trigger email empresa contratada**: se `notify_contractor=true` e existe ClosedDeal ativo com empresa para aquele escopo/equipamento, backend dispara email transacional via Resend (não via Twilio no M1)
- **Modal anti-erro R1**: obrigatório — "Você vai publicar no Condomínio {nome} — confirma?"
