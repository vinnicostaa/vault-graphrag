# S3 — Cadastrar Condomínio (Endereço)

## Finalidade
Etapa 1 de 3 do cadastro: capturar endereço completo, tipo e estrutura física do condomínio (blocos/torres, quantidade de unidades).

## Persona e ABAC
- **Persona primária**: Síndico (`syndic`, role_type: `principal`)
- **Plan tier mínimo**: `trial` (cadastro inicial habilitado durante trial; criação adicional requer tier `plus`/`pro` se passar de 1)
- **ABAC action**: `condominium.create`
- **Restrições**:
  - Apenas síndico principal pode cadastrar condomínio (auxiliary/assistant: bloqueado).
  - RequireTenantMatch ao salvar — gera novo `tenant_id`/`condominium_id`.
  - Hard limit 15 condomínios.

## Fluxo da tela
1. Renderiza wizard com indicador de etapas (1/3).
2. Campo "Nome do condomínio" obrigatório (PDF S3 lista campos).
3. CEP com busca automática (ViaCEP-like) → preenche logradouro, bairro, cidade, UF.
4. Campos manuais: número, complemento (opcional).
5. Campo blocos/torres: toggle "Possui blocos/torres? Sim/Não" → se Sim, lista dinâmica (nome, qtd unidades por bloco).
6. Combo "Tipo de condomínio" (lista mestre fechada): Residencial · Comercial · Misto · Shopping Center · Galeria Comercial · Edifício Garagem.
7. Campo "Quantidade total de unidades" (obrigatório).
8. Se Misto: dois campos opcionais "Unidades residenciais" + "Unidades comerciais".
9. Upload "Foto da fachada" (opcional).
10. Botões: "Avançar" → S4 (mantém estado em wizard) · "Cancelar"/"Voltar".

## Componentes
- `WizardStepper` (3 etapas) — link `[[../../design-system]]`
- `CEPField` (busca + auto-fill)
- `Select` (tipo de condomínio — fechado)
- `RepeatableField` (blocos/torres com nome + qtd)
- `NumberField` (quantidade total)
- `FileUpload` (foto fachada — opcional, jpg/png)
- `WizardActions` (Avançar | Cancelar)

## Estados
- **Loading**: spinner ao carregar wizard se em modo edição.
- **Empty**: campos vazios (default novo cadastro) — sem CTA extra.
- **Error**: validação inline (CEP inválido, quantidade negativa); toast em erro de upload.
- **Success**: persiste rascunho no backend, navega para S4.

## Integração com backend
| Ação | Endpoint | Método | Payload | Retorno |
|---|---|---|---|---|
| Buscar CEP | `/api/v1/utils/cep/{cep}` (inferido) ou ViaCEP externo | GET | — | `{ logradouro, bairro, cidade, uf }` |
| Salvar rascunho etapa 1 | `/api/v1/condominiums/draft` (inferido) | POST | `{ name, address, type, total_units, blocos[], foto_url? }` | `{ draft_id }` |
| Upload foto | `/api/v1/uploads` (inferido) | POST multipart | `file` | `{ url }` |

## Regras de negócio críticas
- Tipo de condomínio é **lista mestre fechada** (não permite "outro").
- "Quantidade total de unidades" obrigatória — base para validação posterior de cadastro de moradores.
- Se Misto e operador informa unidades residenciais + comerciais, soma deve bater com total (validação inferida).
- Cadastro fica em `draft` até concluir S5 (geração de ID/QR).
- Auto-geração ao concluir as 3 etapas: ID alfanumérico seguro + QR Code + link app (S5).
