# S20 — Connect Me Recebido (M→S)

## Finalidade
Caixa de entrada do síndico para Connect Me enviados por moradores (`Morador→Síndico`). Permite responder/aceitar (revelando contato do morador) ou recusar com motivo estruturado. Único fluxo onde síndico é receptor.

## Persona e ABAC
- **Persona primária**: Síndico (`syndic`)
- **Plan tier mínimo**: `base`
- **ABAC action**: `connect_me.respond` (receptor)
- **Restrições**:
  - Apenas síndico do condomínio do morador remetente vê (RequireTenantMatch).
  - Categorias de Connect Me M→S: dúvida_gestão, solicitação_manutenção, reclamação, sugestão, outro.
  - Morador `base` (free): 0/ano (D-067); morador `pagante`: 2-4/ano dependendo do plan.

## Fluxo da tela
1. Lista de solicitações recebidas (cronológica desc), com filtros por categoria e status (pendente/respondida/recusada/expirada).
2. Cada card: categoria, descrição resumida, data envio, expiração (48h), status.
3. Click → drawer/detalhe com texto completo + botões.
4. Ações:
   - **Tenho interesse** → revela nome/contato do morador + log LGPD + status `accepted`.
   - **Recusar** → modal com motivo estruturado (Sem disponibilidade, Escopo incompatível, Capacidade técnica insuficiente, Outro com texto livre obrigatório).
5. Notificação interna de chegada (R9).

## Integração com backend
| Ação | Endpoint | Método | Payload | Retorno |
|---|---|---|---|---|
| Listar recebidas | `/api/v1/connect-me?receiver=me&flow=morador_to_sindico` | GET | — | `[{ id, sender, categoria, descricao, status, expires_at }]` |
| Aceitar | `/api/v1/connect-me/{id}/respond` | POST | `{ action: accept }` | `{ contact: {...}, status: accepted }` |
| Recusar | `/api/v1/connect-me/{id}/respond` | POST | `{ action: reject, reason_code, reason_text? }` | `{ status: declined }` |

## Regras de negócio críticas
- Contato do morador **só visível após aceitar** (Req 19 critério 3).
- Aceite gera log LGPD (`lgpd_logs`): usuário, data, hora, versão termo, dados liberados (C9).
- Recusa pode ser sem texto se motivo != "Outro"; "Outro" exige texto obrigatório.
- Connect Me unidirecional: síndico não pode iniciar conversa em resposta — apenas mostra aceite + contato.
- Após aceitar, comunicação ocorre via canais externos (email/telefone) e o backend registra apenas o aceite.
