# S8 — Nova Ação do Plano Diretor

## Finalidade
Criar uma nova ação estratégica do Plano Diretor com os campos canônicos: tipo, áreas, natureza, importância, risco, impacto, próxima ação, prazos, responsável.

## Fonte canônica
- `vault/90-archive/from-development-content-2026-04-23/contents/####################TELA DO MORADOR#####(1).md` (Bloco 6, TELA 13 — Nova Ação PD; Bloco 5 — listas mestres)
- `vault/90-archive/master-sindico-pdfs-cliente-originais/JORNADA DO SÍNDICO completa.md` (TELA S8 — Cadastrar Ação PD; listas mestre Área Impactada, Natureza, Nível de Importância)
- `vault/90-archive/from-development-content-2026-04-23/02-Jornadas/Inventario-Telas.md` (S8 PD)

## Persona e ABAC
- **Persona primária**: Síndico (`syndic`, principal ou usuário com toggle "Criar/editar Plano Diretor")
- **Plan tier mínimo**: `trial`
- **ABAC action**: `master_plan.create`
- **Restrições**:
  - Permissão granular (matriz Bloco 4 .ini).
  - RequireTenantMatch.
  - Validação obrigatória: título + natureza (rest pode ser opcional conforme cliente).

## Fluxo da tela
1. Form em coluna única ou 2 colunas com seções.
2. Campos:
   - **Título** (string obrigatório)
   - **Descrição** (voz/texto — R2)
   - **Tipo de atividade** (lista mestre 5.1: Administrativo, Financeiro, DP, Jurídico, Manutenção, Obras, Sanitário, Segurança, Sustentabilidade, Tecnologia, Comunicação, Outro)
   - **Áreas impactadas** (multi-select 5.2: Estrutural, Elétrica, Hidráulica, Sanitária, Segurança, Financeira, Jurídica, Trabalhista, Administrativa, Tecnologia, Ambiental, Patrimonial, Convivência, Reputacional, Operacional, Outros) + lista PDF S8 com áreas físicas (Estrutura predial, Hidráulico, Elétrico, etc. — 26 itens)
   - **Natureza** (lista 5.3 — obrigatória: Preventiva, Corretiva, Emergencial, Estratégica, Obrigação legal, Deliberação assemblear)
   - **Nível de importância** (lista 5.5: Baixa, Moderada, Alta, Crítica, Estratégica)
   - **Risco** (categoria múltipla 5.4 + nível único Baixo/Médio/Alto/Crítico)
   - **Impacto esperado** (multi 5.6)
   - **Próxima ação** (multi 5.7)
   - **Data início** + **Data limite** (datepickers)
   - **Responsável** (opcional — ID do usuário ou texto livre)
   - **Observações internas** (texto, não público — opcional)
3. Botões: "Salvar" · "Voltar".

## Componentes
- `Form` (multi-section) — link `[[../../design-system]]`
- `Select` / `MultiSelect` (listas mestre fechadas)
- `VoiceTextarea` (descrição com STT)
- `DateField` (início + limite)
- `RiskMatrixField` (categoria multi + nível único)
- `UserPicker` (responsável)
- `FormActions` (Salvar | Voltar)

## Estados
- **Loading**: spinner ao salvar.
- **Empty**: form vazio (defaults nos selects = não selecionado).
- **Error**: validação inline (título e natureza obrigatórios; data limite >= início).
- **Success**: persiste, redireciona para [[S10]] (detalhe da ação) ou [[S7]] (lista).

## Integração com backend
| Ação | Endpoint | Método | Payload | Retorno |
|---|---|---|---|---|
| Criar ação PD | `/api/v1/condominiums/{id}/master-plan/actions` (inferido) | POST | `{ titulo, descricao, tipo, areas[], natureza, importancia, risco_categorias[], risco_nivel, impacto[], proxima_acao[], data_inicio, data_limite, responsavel?, observacoes_internas? }` | `{ action_id }` |

## Regras de negócio críticas
- Listas mestre são **fechadas** — não permite "outro" exceto onde explicitamente listado (5.1 e 5.2 têm "Outro").
- Natureza obrigatória — base para % preventiva/corretiva no painel S6.
- Status inicial automático = "Planejada" (lista 5.8).
- Não exclui; apenas editar/ocultar com trilha (R3).
- Cada edição posterior gera versão (R4 + C5 trilha).
- "Observações internas" não são públicas aos moradores.
