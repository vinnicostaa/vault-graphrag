# S9 — Lista Ações PD (com filtros avançados)

## Finalidade
Listagem completa das ações do Plano Diretor com filtros avançados (status, natureza, risco, prazo, área impactada). Permite drill-down para o detalhe de cada ação.

## Fluxo da tela
1. Header + breadcrumb (Central → Plano Diretor → Lista).
2. Barra de filtros (toggle expand):
   - Status (lista 5.8: Planejada, Em andamento, Parcialmente concluída, Concluída, Suspensa, Cancelada, Atrasada, Em análise, Aguardando aprovação, Aguardando proposta, Aguardando validação)
   - Natureza (lista 5.3)
   - Risco — categoria + nível
   - Data limite (range)
   - Área impactada (multi-select)
   - Importância (lista 5.5)
3. Busca por título (input com debounce).
4. Ordenação: prazo limite asc | criada desc | risco desc.
5. Lista paginada (50 itens/página) com colunas: Título · Tipo · Status · Natureza · Risco · Prazo · Responsável.
6. Click na linha → S10 (detalhe).
7. Botão flutuante "Nova ação" → S8.

## Componentes
- `FilterBar` (toggleable, multi-filter chips) — link `[[../../design-system]]`
- `SearchInput` (debounced)
- `SortDropdown`
- `DataTable` (responsive — colapsa em cards no mobile)
- `Pagination`
- `FAB` ("Nova ação")
- `EmptyState` (com filtros ativos: "Nenhuma ação corresponde aos filtros")

## Estados
- **Loading**: skeleton de tabela.
- **Empty**: ilustração "Nenhuma ação encontrada" + CTA "Limpar filtros" ou "Nova ação".
- **Error**: toast + retry.
- **Success**: lista paginada renderizada.

## Integração com backend
| Ação | Endpoint | Método | Payload | Retorno |
|---|---|---|---|---|
| Listar com filtros | `/api/v1/condominiums/{id}/master-plan/actions` (inferido) | GET | `?status=&natureza=&risco_nivel=&prazo_de=&prazo_ate=&area=&q=&sort=&page=&size=` | `{ data: [...], total: n, page: n }` |

## Regras de negócio críticas
- Filtros operam sobre listas mestre fechadas — sempre usar enums canônicos.
- Resultado respeita permissões: usuário sem toggle "Tornar registro público aos moradores" não vê internamente flag de visibilidade morador (irrelevante aqui — PD não é público por default).
- Performance: paginação obrigatória (limite default 50; max 200).
