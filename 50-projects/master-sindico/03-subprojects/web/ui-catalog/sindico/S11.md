# S11 — Linha do Tempo (lista)

## Finalidade
Memória oficial do condomínio: lista cronológica imutável de todas as publicações relevantes da gestão (atividades, registros validados, comunicados, eventos, adendos, resultados de enquetes).

## Persona e ABAC
- **Persona primária**: Síndico (`syndic`)
- **Plan tier mínimo**: `trial`
- **ABAC action**: `timeline.list`
- **Restrições**:
  - RequireTenantMatch.
  - Auxiliary/Assistant veem apenas entradas que tenham permissão (toggle "Tornar registro público aos moradores" diferencia visibilidade).
  - Filtro "público/gestão" disponível ao síndico principal.

## Fluxo da tela
1. Header com título "Linha do Tempo" + indicador de imutabilidade.
2. Filtros: período (range), status, natureza, risco, visibilidade (Público/Gestão), tipo de publicação (atividade, registro_validado, comunicado, evento, adendo, resultado_enquete).
3. Cards cronológicos (mais recente no topo, com paginação infinite scroll).
4. Cada card: tipo + título + data + nível de importância + status + empresa envolvida (se houver).
5. Click → detalhe (modal ou rota) com descrição, fotos/vídeos, anexos públicos.
6. Botão "Novo registro" → S12 (criar atividade).
7. Botão "Histórico" → S14.

## Componentes
- `PageHeader` (título + selo "Imutável") — link `[[../../design-system]]`
- `FilterBar` (período, status, natureza, visibilidade, tipo)
- `TimelineList` (cards cronológicos)
- `TimelineCard` (variantes por tipo)
- `InfiniteScroll`
- `PrimaryButton` ("Novo registro")

## Estados
- **Loading**: skeleton de 3 cards.
- **Empty**: "Nenhuma atividade publicada ainda" + CTA "Publicar primeira atividade".
- **Error**: toast + retry.
- **Success**: lista carregada, scroll infinito.

## Regras de negócio críticas
- **LT é a memória oficial** (R1 PDF). Tudo relevante passa por aqui.
- **INSERT-only**: nenhuma entry pode ser editada ou deletada após persistir; correções via novo registro `adendo` (tipo) com link ao original (R3 + R5).
- Nova publicação sempre dispara modal de confirmação anti-erro (R1 .ini): "Publicar para Condomínio X?".
- Plano Diretor reage automaticamente a publicações vinculadas — atualiza status (R2 PDF).
- Visibilidade `Gestão` esconde da listagem do morador (M6).
