# S2 — Seletor de Condomínios

## Finalidade
Permitir ao síndico selecionar entre seus condomínios ativos e encerrados, ou cadastrar um novo. É o ponto de roteamento multi-condomínio com salvaguarda anti-erro (publicar no condomínio errado).

## Persona e ABAC
- **Persona primária**: Síndico (`syndic`, role_type: `principal` | `auxiliary` | `assistant`)
- **Plan tier mínimo**: `trial` (lista vazia até cadastrar primeiro condomínio)
- **ABAC action**: `condominium.list_own` + `condominium.create` (botão)
- **Restrições**:
  - Hard limit: 15 condomínios ativos por síndico (`MaxCondominiumsPerSyndic = 15` — institutional create_condominium_use_case).
  - RequireTenantMatch: lista filtrada por `syndic_id = current_user`.
  - Encerrados aparecem em leitura apenas.

## Fluxo da tela
1. Carrega lista de condomínios via API (`GET /condominiums?owner=me`).
2. Aplica abas: **Ativos** (default) e **Encerrados**.
3. Cada card de Ativo exibe: nome, endereço resumido, tipo (Residencial/Comercial/Misto/Shopping/Galeria/Edifício Garagem), mandato (Eleito/Contratado/Interino + datas), indicadores rápidos (pendências de validação, próximas ações).
4. Cada card de Encerrado exibe: nome, data de encerramento, motivo, badge "Histórico/Leitura".
5. Botão por card (Ativo): "Acessar" → S6 (Central) · "Editar dados" → S3 com pre-fill.
6. Botão por card (Encerrado): "Acessar histórico" (read-only).
7. Botão geral: "Cadastrar novo condomínio" → S3.
8. Botão geral (com permissão): "Encerrar mandato" → fluxo de encerramento (S60 legacy — tratado em sub-produto compliance).

## Componentes
- `TabBar` (Ativos | Encerrados) — link `[[../../design-system]]`
- `CondoCard` (variantes: `active`, `closed`) — exibe pendências em badges
- `IndicatorBadge` (pendências de validação, próximas ações)
- `EmptyState` (quando não há condomínio: CTA "Cadastrar primeiro condomínio")
- `PrimaryButton` ("Cadastrar novo condomínio")
- `SecondaryButton` ("Encerrar mandato")

## Estados
- **Loading**: skeleton de 3 cards.
- **Empty**: ilustração + "Você ainda não administra nenhum condomínio" + CTA cadastro.
- **Error**: toast "Falha ao carregar condomínios" + botão Recarregar.
- **Success**: lista renderizada com ordenação ativos primeiro, depois encerrados (PDF S2 obs devs).

## Integração com backend
| Ação | Endpoint | Método | Payload | Retorno |
|---|---|---|---|---|
| Listar condomínios | `/api/v1/condominiums?owner=me&status=active|closed` (inferido) | GET | — | `[{ id, name, address, type, mandate, indicators }]` |
| Editar condomínio | navega para S3 com `condo_id` | — | — | — |
| Encerrar mandato | `/api/v1/condominiums/{id}/end-mandate` (inferido) | POST | `{ motivo, data_efetiva, observacoes }` | `{ mandate: closed }` |

## Regras de negócio críticas
- **Hard limit 15** — botão "Cadastrar" desabilitado quando atingir o teto (PDF S2 obs devs + decisão D-011).
- Inconsistência histórica: PDF Mar/2026 cita "12 condomínios"; decisão atual = 15. Implementar 15 (ver `quebras-detectadas` Q30).
- Card de Encerrado mantém botão "Acessar histórico" (R6 continuidade institucional).
- Ordenação default: ativos > encerrados, cada grupo por `updated_at desc`.
- Antes de qualquer publicação posterior, modal "Confirmar contexto" (R1 anti-erro) é renderizado.
