# M1 — Painel do Morador

## Finalidade
Home do morador autenticado já vinculado a um condomínio. Apresenta a mensagem institucional de boas-vindas e os 4 cards principais que dão entrada nas jornadas-chave (Linha do Tempo, Plano Diretor, Eventos, Votação de Fornecedor).

## Persona e ABAC
- **Persona primária**: Morador (titular ou dependente, role: `resident`)
- **Plan tier mínimo**: any (base/free liberado para leitura)
- **ABAC action**: `condominium.dashboard.read`
- **Restrições**: exige `Membership.status = active` para algum condomínio. Sem vínculo válido → redireciona para M3 (Buscar Condomínio).

## Fluxo da tela
1. Sistema carrega o condomínio default do morador (último acessado ou único vínculo ativo).
2. Sistema renderiza topo fixo com Nome do condomínio + ID do condomínio.
3. Sistema exibe mensagem institucional: "Bem-vindo ao Painel do Morador. Aqui você acompanha as ações, participa das decisões e acessa as informações oficiais do seu condomínio."
4. Sistema renderiza 4 cards: Linha do Tempo (M6), Plano Diretor (M8), Eventos (M10), Votação de Fornecedor.
5. Usuário clica num card → navega para a tela correspondente.
6. Se usuário tem mais de um vínculo → botão "Trocar condomínio" abre M2.

## Componentes
- `HeaderCondominio` (nome + ID + botão trocar)
- `WelcomeBanner` (mensagem institucional fixa)
- `CardGrid` (4 cards quadrados, ícone + título + contador opcional de pendências)
- `BottomNav` (atalhos para Meu Vínculo M13 e Avaliar Gestão M12)
- Ver [[../../design-system]] para tokens e variantes.

## Estados
- **Loading**: skeleton dos 4 cards + header
- **Empty**: morador sem vínculo ativo → CTA "Adicionar acesso a condomínio" (vai para M3). Mensagem: "Você ainda não está vinculado a nenhum condomínio."
- **Error**: toast "Não foi possível carregar o painel" + botão Tentar novamente
- **Success**: cards renderizados; badge de pendência (avaliação bimestral pendente — M12) se aplicável
