# M7 — Detalhe da Publicação LT

## Finalidade
Exibe detalhe completo de uma TimelineEntry publicada para morador: descrição, fotos, vídeos, anexos públicos, impacto esperado, próxima ação. Se a entry tiver pergunta aos moradores, exibe form de resposta (Sim/Não/Não sei).

## Persona e ABAC
- **Persona primária**: Morador (titular ou dependente)
- **Plan tier mínimo**: any
- **ABAC action**: `timeline.read.detail` + opcional `poll.respond`
- **Restrições**: só TimelineEntry com `visibility=public` é acessível.

## Fluxo da tela
1. Sistema carrega TimelineEntry por id (vindo de M6).
2. Sistema renderiza header: tipo, data, nível de importância, status.
3. Sistema renderiza corpo:
   - Descrição completa
   - Área impactada (chips)
   - Natureza da atividade (Preventiva/Corretiva/Emergencial/Estratégica)
   - Impacto esperado (se autorizado pelo síndico)
   - Próxima ação (se autorizado)
   - Empresa responsável (se houver) com badge verificada/não verificada
   - Período de execução
   - Galeria de fotos / vídeos / anexos (apenas públicos)
4. Se entry tem pergunta vinculada: exibe form com opções Sim/Não/Não sei + botão Enviar resposta.
5. Após enviar resposta: confirma "Resposta registrada" e desabilita form.
6. Voltar → retorna a M6.

## Componentes
- `EntryHeader` (tipo, data, badges)
- `EntryBody` (campos estruturados)
- `MediaGallery` (fotos/vídeos com lightbox)
- `AttachmentList` (anexos PDFs)
- `EmpresaBadge` (verificada/não verificada)
- `PollResponseForm` (radios + submit) — condicional
- Ver [[../../design-system]]

## Estados
- **Loading**: skeleton header + 3 paragraphs
- **Empty**: N/A (entrar nesta tela exige id existente)
- **Error**:
  - 404: "Publicação não encontrada ou removida"
  - 403: "Você não tem acesso a esta publicação" (entry virou private)
- **Success**: conteúdo renderizado; se respondeu → poll form bloqueado com checkmark

## Integração com backend
| Ação UI | Endpoint | Método | Payload | Retorno |
|---|---|---|---|---|
| Carregar detalhe | `/api/v1/institutional/timeline/{id}` | GET | - | TimelineEntry full |
| Listar adendos vinculados | `/api/v1/institutional/timeline/{id}/addenda` | GET | - | Lista[Addendum] |
| Responder pergunta | `/api/v1/institutional/polls/{poll_id}/responses` | POST | { option } | PollResponse |

Endpoints inferidos. Adendos podem vir embutidos no GET principal.

## Regras de negócio críticas
- "Somente conteúdo liberado pelo síndico" — fields como Impacto esperado / Próxima ação só aparecem se síndico marcou como público (S15)
- Adendos aparecem como **entries separados** (R3, R5) vinculados; nunca substituem o original
- Pergunta aos moradores: 1 resposta por morador, imutável após envio (R5 cross-domain pós-fechamento)
- Anexos respeitam ABAC (downloadáveis ou só preview, conforme política)
- Vídeos institucionais de empresa: não exibidos aqui — apenas em assembleia de escolha de fornecedor (Decision Marketing)
