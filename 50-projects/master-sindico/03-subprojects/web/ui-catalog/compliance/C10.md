# C10 — Score Interno de Governança

## Finalidade
Exibir, **privadamente** ao síndico, um score que mede qualidade da gestão (governança + compliance). Acompanha gamificação profissional + recomendações automáticas para regularização. **Não** é exposto a moradores nem a terceiros.

## Fonte canônica
- `vault/90-archive/from-development-content-2026-04-23/03-Modulos/Compliance-Detalhado.md` §C10
- `backend/.kiro/specs/master-sindico/requirements/cross-domain.md` Req 33 + Req 41

## Persona e ABAC
- **Persona primária**: Síndico principal (privado — só ele acessa)
- **Plan tier mínimo**: `plus` (sugerido)
- **ABAC action**: `compliance.score.read.self`
- **Restrições**: **PRIVADO**. Nenhum outro perfil (preposto, administradora, conselho, morador) vê. Admin global pode ver para suporte.

## Variáveis do cálculo (canônico)
- Percentual de ações preventivas
- Cumprimento de prazos
- Classificação de risco adequada (cobertura)
- Participação dos moradores
- Compliance ativo
- Validações em dia
- Trilha consistente (sem buracos na auditoria)

## Exibição
- **Score Governança** (1-10) — ver [[../../../../01-domain/invariants#INV-GOV-001]]
- **Score Compliance** (0-100) — ver [[../../../../01-domain/invariants#INV-GOV-002]]

>
> **Distinção**:
> - **C10 (esta tela)** — Score Compliance **privado** do síndico (breakdown C2+C3+C4+C5+C6 do próprio módulo). Visibilidade: apenas o síndico + admin MS.
> - **[[../sindico/S32-score-governanca|S32]]** — Perfil público do síndico com **2 scores** visíveis (Governança 1-10 + Compliance 0-100 agregado). Visibilidade: morador do mesmo condomínio, empresa via Connect Me, qualquer um em perfil público.
> - Scores são os **mesmos valores canônicos** (INV-GOV-001/002) — diferença é **escopo de visibilidade** + **detalhamento** (C10 mostra breakdown intrusivo, S32 só agregado).

## Recomendações automáticas (canônicas)
- Regularizar assinaturas ([[C3]])
- Classificar risco pendente ([[C7]])
- Atualizar ação vencida
- Encerrar votação aberta

## Componentes
- `ScoreGauge` (medidor circular ou linear)
- `ScoreTrendChart` (evolução temporal)
- `RecommendationList` (cards acionáveis com link direto para a tela respectiva)
- `LockedFeature` (se tier < plus): exibe teaser e CTA upgrade
- `PrivacyBadge` ("Privado — visível apenas para você")

## Estados
- **Loading**: skeleton dos gauges.
- **Empty (mandato muito recente)**: "Score precisa de pelo menos 30 dias de dados para calcular."
- **High score**: badge celebrativa "Excelente gestão!".
- **Low score**: banner com recomendações em destaque.
- **Tier < plus**: tela bloqueada com CTA "Faça upgrade para Plus para ver seu score".
- **Error**: toast com retry.

## Integração com backend
| Ação | Endpoint (inferido) | Método | Retorno |
|---|---|---|---|
| Score atual | `/api/v1/compliance/score` | GET | `{ governance_score, compliance_score, recommendations[] }` |
| Histórico | `/api/v1/compliance/score/history` | GET | Série temporal |

## Regras de negócio críticas
- **Privado** — qualquer exposição externa exige nova decisão (não é o caso atual).
- Recálculo: job diário (sugerido) ou on-demand quando muda evento crítico.
- Não há "ranking" entre síndicos (justamente porque é privado).
- Recomendações apenas comunicam, nunca corrigem automaticamente.
- Gamificação profissional: badges/níveis ficam em backlog M4+.
