# C5 — Trilha de Auditoria Visível

## Finalidade
Listar de forma estruturada todos os eventos rastreáveis da gestão (criação de atividades, validações, publicações, alterações cadastrais, login/logout, votações, etc.). Base probatória append-only.

## Fonte canônica
- `vault/90-archive/from-development-content-2026-04-23/03-Modulos/Compliance-Detalhado.md` §C5
- `backend/.kiro/specs/master-sindico/requirements/cross-domain.md` Req 37 (append-only)
- `backend/.kiro/STATE.md` DT-010 (`IAuditLogger` cross-módulo)

## Persona e ABAC
- **Persona primária**: Síndico principal + Admin Master Síndico
- **Plan tier mínimo**: `trial`
- **ABAC action**: `compliance.audit.read`
- **Restrições**: **apenas admin + síndico principal** acessam (síndico preposto, administradora e conselho NÃO veem).

## Colunas da lista
| Coluna | Descrição |
|---|---|
| Usuário | Nome + avatar |
| Perfil | Role (sindico/preposto/administradora/conselho/admin) |
| Ação realizada | Texto descritivo (ex: "atualizou plano diretor") |
| Campo alterado | Nome do campo afetado |
| Versão anterior | Snapshot pré-mudança |
| Versão atual | Snapshot pós-mudança |
| Data · Hora | ISO 8601 + timezone do condomínio |

## Filtros
- Usuário (autocomplete)
- Data (range picker)
- Tipo de ação (multi-select)

## Eventos rastreados (canônico — Bloco 18 + Req 37)
- Criação de atividades
- Validação de registros
- Publicação de comunicados
- Alterações cadastrais
- Atualização do plano diretor
- Login/logout
- Criação/modificação de pauta de assembleia
- Votação
- Mudança de permissões (membership)
- Exclusão de dados (soft delete — registra, não some)

## Componentes
- `AuditTable` (paginada server-side, sortable)
- `FilterBar` (Usuário + Data + Tipo)
- `DiffView` (modal "Ver diferença" comparando versões anterior/atual)
- `ExportButton` ("Exportar CSV/JSON" — gera evento `audit.exported`)

## Estados
- **Loading**: skeleton de 10 linhas.
- **Empty (filtro restritivo)**: "Nenhum evento encontrado com esses filtros."
- **Empty (geral)**: improvável — sempre há login/logout.
- **Error**: toast com retry; preserva filtros aplicados.
- **Acesso negado**: 403 com texto "Acesso restrito ao síndico principal e admin."

## Integração com backend
| Ação | Endpoint (inferido) | Método | Retorno |
|---|---|---|---|
| Listar eventos | `/api/v1/compliance/audit-trail` | GET | Paginado |
| Filtros disponíveis | `/api/v1/compliance/audit-trail/filters` | GET | Lista de tipos |
| Exportar | `/api/v1/compliance/audit-trail/export` | POST | URL signed |

## Regras de negócio críticas
- **Append-only / imutável** (Req 37): nunca se edita ou apaga linha; correção apenas via adendo ([[C6]]).
- **Retenção 5 anos** (LGPD).
- **Archive em MinIO/S3 Glacier após 1 ano** (estratégia de hot/cold storage).
- **Soft delete registra** — quando usuário "exclui" algo, gera linha de auditoria, não some do banco.
- Toda exportação gera evento próprio (auditoria recursiva).
- Implementação: `IAuditLogger` cross-módulo (DT-010 STATE — adapter centralizado em Sprint 10).
