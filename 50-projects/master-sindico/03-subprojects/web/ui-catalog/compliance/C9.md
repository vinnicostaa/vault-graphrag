# C9 — Registro LGPD

## Finalidade
Listar histórico de compartilhamento de dados via Connect Me e demais módulos: Empresa contatada, Data, Finalidade, Dados compartilhados, Consentimento registrado. Proteção contra alegação futura de vazamento. Imutável.

## Fonte canônica
- `vault/90-archive/from-development-content-2026-04-23/03-Modulos/Compliance-Detalhado.md` §C9
- `backend/.kiro/specs/master-sindico/requirements/cross-domain.md` Req 39 (`lgpd_logs`)

## Texto fixo (literal)
> "Os dados foram compartilhados mediante autorização expressa para finalidade específica, conforme Lei 13.709/2018."

## Persona e ABAC
- **Persona primária**: Síndico principal + Admin Master Síndico
- **Plan tier mínimo**: `trial`
- **ABAC action**: `compliance.lgpd.read`

## Colunas
| Coluna | Tipo |
|---|---|
| Empresa contatada | Texto + link perfil |
| Data | ISO + timezone |
| Finalidade | Texto curto (`finalidade` selecionada no Connect Me) |
| Dados compartilhados | Lista (campos do `consent_payload`) |

| IP origem | IPv4/v6 |

## Implementação atual (canônica)
- Tabela central: **`lgpd_logs`** integrada com **todos** os módulos (Connect Me, Banco de Talentos, Vizinhança, etc.).
- **Retenção: 5 anos**.
- **Direito ao esquecimento**: purga **após** vencimento (5 anos).

## Componentes
- `LGPDLogTable` (paginada server-side)
- `FilterBar` (Empresa · Período · Finalidade)
- `LegalDisclaimer` (rodapé com texto fixo da Lei 13.709/2018)
- `ExportButton` ("Exportar relatório LGPD" — gera PDF com cabeçalho jurídico)
- `RecordDrawer` (detalhe completo do consentimento)

## Estados
- **Loading**: skeleton de 8 linhas.
- **Empty**: "Nenhum compartilhamento registrado."
- **Filtrado**: contagem visível ("Exibindo X de Y").
- **Error**: toast com retry.

## Integração com backend
| Ação | Endpoint (inferido) | Método | Retorno |
|---|---|---|---|
| Listar logs | `/api/v1/compliance/lgpd-logs` | GET | Paginado |
| Detalhe consentimento | `/api/v1/compliance/lgpd-logs/{id}` | GET | Item completo |
| Exportar relatório | `/api/v1/compliance/lgpd-logs/export` | POST | URL signed |

## Regras de negócio críticas
- **Imutável** — nenhum log pode ser editado/deletado durante a retenção.
- Trigger: tabela `lgpd_logs` recebe insert via evento de domínio em CADA módulo que compartilhe dados pessoais (Connect Me Req 19, Banco de Talentos Req 28, etc.).
- **Direito ao esquecimento**: solicitação do titular é registrada como **novo** log; purga real só após 5 anos da última atividade.
- Sem permissão de delete manual (nem para admin); purga é job automatizado.
- Cross-validation: cada log deve referenciar um `consent_payload` válido (verificação backend).
