# C3 — Assinaturas Obrigatórias dos Usuários

## Finalidade
Gerenciar e exibir o status de assinatura do termo institucional pelos demais usuários autorizados a operar a gestão (até 3 perfis convidados além do síndico principal). Compliance só fica **"Em dia"** quando TODOS assinam.

## Texto jurídico (literal) para usuários convidados
> "Declaro ciência de que minha atuação nesta plataforma é rastreável e deve refletir informações verdadeiras e diligentes. Estou ciente de que minhas ações poderão compor histórico documental da gestão condominial."

## Persona e ABAC
- **Persona primária (visualização)**: Síndico principal (vê lista completa)
- **Persona secundária (assinatura)**: Síndico preposto, Administradora, Conselheiro (assinam o próprio termo)
- **Plan tier mínimo**: `trial`
- **ABAC action**: `compliance.user_signature.read` (visualizar) · `compliance.user_signature.sign` (assinar próprio)

## Lista exibida
Até 3 usuários cadastrados além do principal:
- Síndico principal (referência)
- Síndico preposto
- Administradora
- Conselho (representante)

Para cada linha: **nome · perfil · status (Assinado/Pendente) · data se assinado · botão Reenviar convite (se Pendente)**.

## Componentes
- `UserSignatureRow` (avatar, nome, perfil, status badge, ação)
- `BulkResendButton` ("Reenviar convite a todos os pendentes")
- `LegalTextModal` (mostra texto ao usuário ao clicar Assinar)
- `SignatureProgress` (X/Y assinados — barra visual)

## Estados
- **Loading**: skeleton de 4 linhas.
- **Empty**: caso não haja convidados — CTA "Convidar usuário" linkando para gestão de membership.
- **Mixed**: lista padrão com badges Pendente/Assinado.
- **All signed (Em dia)**: banner verde "Todas as assinaturas em dia" + bloqueio de alterações sem nova ação admin.
- **Error**: toast em falha de carregamento, mantém estado anterior.

## Integração com backend
| Ação | Endpoint (inferido) | Método | Payload |
|---|---|---|---|
| Listar status | `/api/v1/compliance/signatures` | GET | — |
| Reenviar convite | `/api/v1/compliance/signatures/{user_id}/resend` | POST | — |
| Assinar (pelo próprio usuário) | `/api/v1/compliance/signatures/sign` | POST | `{ terms_version, accepted: true }` |

## Regras de negócio críticas
- **Compliance só fica "Em dia" quando TODOS assinam** (regra dura — derivada para [[C1]]).
- Cada assinatura individual gera publicação automática imutável na Linha do Tempo.
- Usuário convidado precisa ter sessão Zitadel ativa para assinar.
- Não pode haver "delegação" de assinatura — cada usuário assina por si.
- Reenvio de convite gera evento auditável.
- Continuidade entre mandatos: usuários antigos preservam histórico; novos precisam assinar.
