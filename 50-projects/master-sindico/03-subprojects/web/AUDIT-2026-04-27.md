## 2. Findings — 26 itens catalogados

### 🔴 Críticos M1-bloqueantes (3)

#### A-WEB-2026-04-27-001 · `apps/admin` (port 3010) ausente

- **Vault declara**: D-134 (2026-04-25, revoga D-072) cria `apps/admin` dedicada com 5 módulos (Dashboard Financeiro, Gestão Usuários, Moderação, Configuração Plataforma, Broadcast) atrás de Cloudflare Access SSO+MFA.
- **Realidade**: pasta `apps/admin/` não existe; vault `03-subprojects/web/_moc.md` lista mas código não está scaffolded.
- **Impacto M1**: M1 exige "esqueleto admin" segundo `03-subprojects/web/_moc.md` linha 156.
- **Mitigação**: scaffold mínimo idêntico aos demais 5 apps base + railway.json + porta 3010 + `_layout.tsx` com guard `requireRole('platform_admin')` placeholder + 1 placeholder route.
- **Esforço**: 4h (scaffold) + 8h (route guards + UI banner Modo Admin).

#### A-WEB-2026-04-27-002 · OTP code passado via search params (vazamento real)

- **Onde**: `forgot-password/code.tsx` linha 60 → `forgot-password/new-password.tsx` recebe `code` em search; `verify-email.tsx` linha 65 → `set-password.tsx` recebe `code` em search.
- **Vetor**: códigos 6 dígitos vazam em histórico do browser, `Referer` headers, logs de proxy/CDN, eventualmente compartilhamento via URL/screenshot.
- **STEERING #18** + **vault security.md §4** exigem dados sensíveis fora do URL.
- **Mitigação canônica**: backend retorna `reset_token` (JWT/opaco TTL 5min) no `confirm-reset-code` / `verify-email`; frontend passa em navigation state ou cookie efêmero. Schema `ResetPasswordSchema` recebe `reset_token` em vez de `code`.
- **Coordenação**: requer mudança contratual com backend (`/api/v1/auth/confirm-reset-code` → emitir token) — abrir D-### + ADR.

#### A-WEB-2026-04-27-003 · PKCE gap — auth-context faz username/password POST direto

- **STEERING #18** + **D-001**: "PKCE obrigatório, code_verifier em cookie gorilla/securecookie".
- **Realidade**: `apps/auth/src/contexts/auth-context.tsx` linhas 41-49 faz `POST /api/v1/auth/sign-in` com `{email, password}` — **não é PKCE**. Tela `/callback` cobre PKCE return, mas `/sign-in` não redireciona para Zitadel hosted login.
- **Inconsistência**: `oidc-client-ts ^3.5.0` instalado mas não importado em código.
- **3 cenários possíveis**:
  1. Backend tem 2 fluxos: local creds (dev) + OIDC PKCE (prod). Frontend usa o errado.
  2. Backend só tem OIDC PKCE. `/sign-in` é stub que vai ser removido.
  3. Backend implementa hybrid (Resource Owner Password Credentials + OIDC). RFC 6749 deprecou ROPC.
- **Próximo passo**: validar contrato `/api/v1/auth/sign-in` no backend Go (`internal/modules/identity/infrastructure/http/routes/`). Decisão D-### em STATE.

### 🟠 High (10) — corrigir antes de M1

#### A-WEB-2026-04-27-004 · btn-destructive shortcut undefined (DT-WEB-002 confirmado)

- `packages/ui/src/button.tsx` linha 14 mapeia variant `destructive: "btn-destructive"` mas `uno.config.ts` (linhas 168-189) não define o shortcut.
- Resultado: botão destrutivo fica com cor padrão (não vermelha).
- **Fix**: adicionar `["btn-destructive", "btn-base bg-destructive text-destructive-foreground hover:bg-destructive/90"]` em todos 6 `uno.config.ts` (ou no preset compartilhado quando criado).

#### A-WEB-2026-04-27-005 · Scrollbar usa var(--color-border) (DT-WEB-003 confirmado)

- `apps/auth/src/main.css` linhas 163-167 usa `var(--color-border)` e `var(--color-muted-foreground)`. CSS vars reais são `--border` e `--muted-foreground`.
- Resultado: scrollbar fica `transparent` (fallback) — efetivamente invisível.
- **Fix**: trocar para `var(--border)` e `var(--muted-foreground)` nos 6 `main.css`.

#### A-WEB-2026-04-27-006 · uno.config.ts duplicado byte-a-byte 6× (DT-WEB-004 confirmado)

- 6 arquivos `apps/<app>/uno.config.ts` com 8458 bytes idênticos.
- Drift inevitável quando alterar tema em produção.
- **Fix**: extrair para `packages/ui/src/uno.preset.ts` exportando função; cada app importa `import { masterSindicoPreset } from "ui/uno.preset"; presets: [...masterSindicoPreset()]`.
- **FE-017** já estava no roadmap M1.

#### A-WEB-2026-04-27-007 · createEffect+setInterval anti-pattern (verify-email + forgot-password/code)

- Confirmado via SolidJS docs oficiais (Context7): pattern correto é `setInterval` + `onCleanup` **DENTRO** do `createEffect`. Atual: interval declarado fora, onCleanup separado fora.
- Risco: se effect re-rodar (lendo signal reativo), interval anterior nunca limpa → vazamento.
- **Atual no projeto**: effect não lê signals reativos; bug é teórico mas anti-pattern.
- **Fix**: usar `onMount` (executa 1x) ou refatorar para colocar `onCleanup` dentro do `createEffect`.

#### A-WEB-2026-04-27-008 · ServerErrorPage exibe error.message direto

- `apps/auth/src/components/error-pages.tsx` linha 55: exibe `props.error.message` raw para o usuário.
- Em produção, mensagens podem revelar stack traces, paths, SQL fragmentos.
- **Fix**: condicional `import.meta.env.PROD ? "Erro interno (referência {requestId})" : message()` + sempre logar Sentry com requestId.

#### A-WEB-2026-04-27-009 · AuthSession não validado com Zod no auth-context

- **vault security.md §3** + **CLAUDE.md vault §10**: "Validar Zod toda resposta backend".
- `apps/auth/src/contexts/auth-context.tsx` linha 33: `res.data?.data ?? null` direto, tipado como `AuthSession`. Sem `AuthSessionSchema.parse()`.
- Backend pode regredir/bug e frontend não detecta.
- **Fix**: importar `AuthSessionSchema` de `schemas`, `safeParse`, log Sentry em mismatch.

#### A-WEB-2026-04-27-010 · Biome a11y: 6 errors + 14 warnings em apps/auth

- `bun run check` exit code 1.
- Confirmados: `useFocusableInteractive` em sign-up.tsx:83 (span role=tab sem tabIndex), `useSemanticElements` em sign-in.tsx:135 (role=group → fieldset).
- Bloqueia DoD #1 ("biome sem warnings/errors").
- **Fix**: substituir spans interativos por `<button type="button">` ou adicionar `tabIndex={0}`; trocar `<div role="group">` por `<fieldset>`.

#### A-WEB-2026-04-27-011 · Confusão `bun test` vs `bun run test` quebra suite UI/auth

- `bun test` (Bun runner próprio) ignora `vitest.config.ts` e roda sem jsdom → `ReferenceError: document is not defined` em UI tests.
- `bun run test` invoca o script `"test": "vitest run"` corretamente.
- **Fix**: documentar no CLAUDE.md raiz e em cada README de package + alias scripts para evitar ambiguidade. Confirmar com `bun run test` que suite passa.

#### A-WEB-2026-04-27-012 · Vault `03-subprojects/web/README.md` desatualizado

- README diz `apps/auth: scaffold + sign-in/sign-up stub` — atualmente tem 13 rotas + AuthProvider + MSW + 5 test files.
- README diz `packages/schemas: vazio (stub console.log)` — atualmente tem auth.ts + common.ts + 30+ test cases.
- README diz `packages/ui: apenas Button` — atualmente 7 componentes.
- DT-WEB-005 marcado como pendente mas resolvido.
- **Fix**: atualizar README + _moc + tasks.md no vault para refletir realidade. Marcar DT-WEB-005 como 🟢 resolvido.

#### A-WEB-2026-04-27-013 · 5 apps base têm código morto + ColorModeProvider sem initialColorMode + routes/index.tsx duplicado

- `apps/{cms,lms,forum,assembly,lp}/src/index.tsx` têm `Solid.createEffect(() => router.update({}))` morto, AuthProvider/QueryClientProvider apenas comentados, ColorModeProvider sem `initialColorMode` (auth usa "dark").
