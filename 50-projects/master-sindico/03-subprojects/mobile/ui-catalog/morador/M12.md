# M12 — Avaliar Gestão (Mobile)

## Finalidade

Avaliação bimestral obrigatória da gestão do síndico — 3 critérios 1-10. Feeds score Reputação (D-051). M2 no mobile.

## Persona e ABAC

- Morador, `assessment.submit`.
- Período elegível = bimestre corrente (derivado server-side).

## Fluxo mobile

1. Entrada: badge home ou push silent início ciclo.
2. Tela exibe 3 sliders verticais (Transparência / Eficácia / Comunicação) com labels semânticos.
3. Campo opcional texto (max 500) "Comentário (opcional)".
4. CTA "Enviar avaliação" → modal confirmação (aviso irreversível).
5. `POST /reputation/assessment` → tela agradecimento + botão voltar home.

## Widget Flutter principal

```dart
class AvaliarGestaoPage extends StatelessWidget {
  Widget build(BuildContext context) {
    return BlocConsumer<AssessmentBloc, AssessmentState>(
      listener: (ctx, s) => s.whenOrNull(done: () => context.pop()),
      builder: (ctx, s) => Scaffold(
        appBar: AppBar(title: Text('Avaliar gestão')),
        body: _Form(),
        bottomNavigationBar: _SubmitButton(),
      ),
    );
  }
}
```

- 3 `Slider` widgets com `divisions: 9` (1..10).
- `TextField` multiline comment.
- Bloc emit `Submitting` → botão loading; `Done` → pop + snackbar "Obrigado pela avaliação".

## Platform-specific notes

- iOS: haptic medium após submit confirm.
- Android: back button durante submit = dismiss dialog sem cancelar request.
- **A11y**: sliders com `Semantics(value: '7 de 10')`; volume-up/down ajustam valor.

## Estados

- Loading / Idle / Submitting / Done / OutOfWindow / AlreadySubmitted / Error.

## Offline behavior

- Avaliação **online only**. Se offline → banner "Conecte-se para enviar" + botão desabilitado.
- Não persistir rascunho (risco de vazamento).

## Push notification trigger

- Recebe `assessment.period.opened.{user_id}` no início bimestre (silent data-only).
- Notificação visível 15 dias antes do fim do período.

## Integração backend

| Ação | Endpoint | Método |
|---|---|---|
| Status pendência | `/api/v1/reputation/assessment/pending` | GET |
| Enviar | `/api/v1/reputation/assessment` | POST |

## Gaps/ressalvas

- Tela de "histórico pessoal" (M14) só em M3.
- Se condomínio tem > 1 síndico, UI exige seletor — hoje assumimos 1 síndico por bimestre.
