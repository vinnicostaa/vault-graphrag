# M11 — Comunicados (Mobile)

## Finalidade

Lista de comunicados oficiais do condomínio. Read-only no mobile M1. Push notification + deep-link para item.

## Persona e ABAC

- Morador, `announcement.read`.

## Fluxo mobile

1. Card M1 "Comunicados" → lista.
2. Deep-link `mastersindico://announcements/{id}` via push → abre lista + auto-push detail.
3. Filtro chip prioridade (M2): todos / urgentes / normais.
4. Tap item → detail full-screen com markdown.

## Widget Flutter principal

```dart
class ComunicadosPage extends StatelessWidget {
  Widget build(BuildContext context) {
    return BlocBuilder<AnnouncementsBloc, AnnouncementsState>(
      builder: (ctx, state) => Scaffold(
        appBar: AppBar(title: Text('Comunicados')),
        body: RefreshIndicator(
          onRefresh: () async => ctx.read<AnnouncementsBloc>().add(RefreshRequested()),
          child: ListView.separated(...),
        ),
      ),
    );
  }
}
```

## Platform-specific notes

- iOS: haptic heavy ao receber urgente em foreground.
- Android: badge counter no launcher icon (via `flutter_app_badger`).
- **SafeArea** + tamanho mínimo 16sp body.

## Estados

- Loading / Loaded / Empty / Error / Offline / UrgentBanner (se há urgente não lido topo).

## Offline behavior

- Hive 30min TTL lista, 7d detail.
- Push invalida cache + incrementa badge.

## Push notification trigger

- `announcement.{cond}.new` — canal `ms_general`.
- `announcement.{cond}.urgent` — canal `ms_urgent` + `time-sensitive` iOS.
- Tap → deep-link `/announcements/{id}`.

## Componentes

- `AnnouncementTile(priority, title, preview, unread)`.
- `UrgentBanner` (topo lista) se há urgente não lido.

## Integração backend

| Ação | Endpoint | Método |
|---|---|---|
| Lista | `/api/v1/institutional/announcements/{cond}` | GET |
| Detalhe | `/api/v1/institutional/announcements/{id}` | GET |
| Mark read | `/api/v1/institutional/announcements/{id}/read` | POST |

## Gaps/ressalvas

- "Marcar todos como lido" — backlog M2.
- Anexo grande (> 10MB): só link; download manual.
