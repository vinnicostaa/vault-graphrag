# M10 — Eventos (Mobile)

## Finalidade

Lista de eventos do condomínio. Tab bottom nav "Eventos". Acessa detalhe + RSVP (M2+).

## Persona e ABAC

- Morador, `events.read`.

## Fluxo mobile

1. Tab "Eventos" → lista `GET /institutional/events/{cond}?range=upcoming`.
2. Cada item: bloco data (dia/mês grande) + título + local + chip status.
3. Tab secundária "Passados" → `range=past`.
4. Tap → M10-DETAIL com RSVP.

## Widget Flutter principal

```dart
DefaultTabController(
  length: 2,
  child: Scaffold(
    appBar: AppBar(
      title: Text('Eventos'),
      bottom: TabBar(tabs: [Tab(text: 'Próximos'), Tab(text: 'Passados')]),
    ),
    body: TabBarView(children: [EventsList(upcoming: true), EventsList(upcoming: false)]),
  ),
);
```

## Platform-specific notes

- iOS: `CupertinoTabBar` alternativo (mas mantém Material para consistência DS).
- Android: badge tab count.
- **Localização pt-BR** obrigatória em datas (intl).

## Estados

- Loading / Loaded / Empty ("Sem eventos próximos") / Offline / Error.

## Offline behavior

- Hive `events_{cond}` 15min TTL.
- RSVP offline (M2+): enfileirar em sync queue.

## Push notification trigger

- `event.{id}.reminder` 1h antes.
- `event.{id}.cancelled` → remove da lista + toast.

## Componentes

- `EventListTile(event)` com data block + body.
- `RsvpBottomSheet` (M2).
