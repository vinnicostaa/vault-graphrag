# M1 — Painel do Morador (Mobile)

## Finalidade

Home do morador autenticado já vinculado a um condomínio. Equivalente mobile de [[../../../web/ui-catalog/morador/M1]] — adaptada para bottom nav, touch targets ≥ 48dp, grade 2x2 de cards.

## Fonte canônica

- [[../../../web/ui-catalog/morador/M1]] (web equivalent)
- `vault/90-archive/master-sindico-pdfs-cliente-originais/JORNADA DO MORADOR.md` (TELA M1)

## Persona e ABAC

- **Persona**: Morador (role `resident`)
- **ABAC action**: `condominium.dashboard.read`
- **Plan tier**: any
- **Pré-condição**: `Membership.status = active`; senão → empty state com CTA `ONB-M2`.

## Fluxo mobile

1. Splash resolve → ShellRoute com bottom nav 4 abas (Home / Timeline / Eventos / Perfil); tab Home = esta tela.
2. Carrega `GET /institutional/memberships/me/default` + `GET /institutional/dashboard/resident` em paralelo.
3. Renderiza header condomínio (AppBar customizado) + grade 2x2 cards.
4. Badge com contador em card quando há pendência.
5. Pull-to-refresh recarrega tudo.

## Widget Flutter principal

```dart
class HomeMoradorPage extends StatelessWidget {
  Widget build(BuildContext context) {
    return BlocBuilder<HomeBloc, HomeState>(
      builder: (context, state) => state.when(
        loading: () => _Skeleton(),
        ready: (membership, dash) => _Content(membership, dash),
        empty: () => _EmptyCTA(),
        offline: (stale) => _OfflineBanner(stale),
        error: (f) => _Error(f),
      ),
    );
  }
}
```

- Bloc: `HomeBloc` (hydrated)
- Cards: `GridView.count(crossAxisCount: 2)` + `AspectRatio(1.0)`
- Cada card: `CardTile(icon, title, badge?)` com `InkWell` onTap → `context.push(...)`.

## Platform-specific notes

- **iOS**: `CupertinoSliverRefreshControl` opcional; SafeArea obrigatória top.
- **Android**: back button no Home = `SystemNavigator.pop()` (minimize app) após confirmação.
- **Both**: AppBar height 56dp; bottom nav 64dp; card min-height 140dp.

## Estados

- **Loading**: 4 skeletons dos cards + header shimmer.
- **Empty**: Ícone + texto "Você ainda não está vinculado a nenhum condomínio." + botão `ONB-M2`.
- **Error**: toast "Não foi possível carregar o painel" + botão "Tentar novamente".
- **Offline**: Banner topo "Você está offline — dados de <X min atrás>" + cards com cache.
- **Success**: cards + badges.

## Offline behavior

- Se cache `dashboard_resident` TTL expirou mas existe → renderiza + banner; fetch em background.
- Se sem cache + sem rede → empty state especial "Sem conexão".
- Pull-to-refresh sempre tenta rede, fallback cache.

## Push notification trigger

- Recebe `timeline.{id}.new` → badge card Timeline incrementa (even if app foreground).
- Recebe `announcement.{id}.new` → badge card Comunicados incrementa.
- Recebe `event.{id}.cancelled` → invalida cache events + badge decrementa.

## Componentes

- `HeaderCondominio` (nome + ID + IconButton trocar)
- `WelcomeBanner` (mensagem institucional fixa)
- `CardGrid` (4 cards)
- `BottomNavScaffold` (shell — ver [[../../../architecture]])

## Integração com backend

| Ação | Endpoint | Método |
|---|---|---|
| Membership default | `/api/v1/institutional/memberships/me/default` | GET |
| Dashboard counters | `/api/v1/institutional/dashboard/resident` | GET |
| Trocar default | `/api/v1/institutional/memberships/me/default` | PATCH |

## Regras de negócio

- R3 PDF Morador: sem vínculo → sem painel (empty state).
- Avaliação bimestral pendente → badge em card Eventos/Comunicados (mobile) ou bloqueio M3.

## Gaps/ressalvas

- Empty state design sem mockup cliente; usar padrão Material illustration.
- "Trocar condomínio" em mobile = bottom sheet (não modal centro como web).
