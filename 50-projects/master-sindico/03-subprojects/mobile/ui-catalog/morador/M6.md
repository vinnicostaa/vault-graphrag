# M6 — Timeline Read (Mobile)

## Finalidade

Lista append-only da Linha do Tempo do condomínio. Morador consome; escrita = web. Tab "Timeline" do bottom nav.

## Persona e ABAC

- **Persona**: Morador (qualquer plano).
- **ABAC action**: `timeline.read`.

## Fluxo mobile

1. Tab Timeline ativa → carrega `GET /institutional/timeline/{cond}?page=1&size=20`.
2. Lista ordenada desc por `published_at`; chip-bar filtro no topo (todos/comunicados/atas/eventos).
3. Scroll próximo do bottom → carrega próxima página.
4. Tap item → M7 (detalhe).
5. Pull-to-refresh → recarrega p1.
6. Primeira visualização do item (scroll na viewport > 2s) marca read.

## Widget Flutter principal

```dart
class TimelinePage extends StatefulWidget {
  State<TimelinePage> createState() => _TimelinePageState();
}

class _TimelinePageState extends State<TimelinePage> {
  final _scroll = ScrollController();

  Widget build(BuildContext context) {
    return BlocBuilder<TimelineBloc, TimelineState>(
      builder: (ctx, state) => RefreshIndicator(
        onRefresh: () async => ctx.read<TimelineBloc>().add(const RefreshRequested()),
        child: CustomScrollView(
          controller: _scroll,
          slivers: [
            SliverAppBar(pinned: true, title: Text('Linha do Tempo')),
            _SliverFilterChips(),
            _SliverListWithSkeleton(state),
          ],
        ),
      ),
    );
  }
}
```

- Bloc: `TimelineBloc` (hydrated)
- Item widget: `TimelineItemTile(entry)`
- Detail route: `context.push('/timeline/${entry.id}')` → M7.

## Platform-specific notes

- **iOS**: `CupertinoSliverRefreshControl` alternativa; haptic feedback light no refresh.
- **Android**: material `RefreshIndicator`.
- **Both**: `PageStorageKey('timeline-list')` preserva scroll ao trocar tab.

## Estados

- **Loading**: 5 item skeletons.
- **Empty**: "Ainda não há registros na linha do tempo."
- **Loaded**: lista + "carregando mais" no final se `hasMore`.
- **Offline**: banner + itens cacheados.
- **Error**: toast retry.

## Offline behavior

- Cache Hive `timeline_{cond}` (TTL 1h, cap 100).
- Detail individual cacheado 7d após primeira leitura.
- Push `timeline.{id}.new` marca stale → refetch p1 no foreground.

## Push notification trigger

- Recebe `timeline.{cond}.new` → incrementa badge tab Timeline + se app foreground adiciona item no topo com animação.

## Componentes

- `TimelineItemTile` (chip categoria + título + preview + data relativa)
- `FilterChipBar`
- `SkeletonList`
- `OfflineBanner` compartilhado

## Integração com backend

| Ação | Endpoint | Método |
|---|---|---|
| Lista paginada | `/api/v1/institutional/timeline/{cond}?page&size&category` | GET |
| Detalhe | `/api/v1/institutional/timeline/{id}` | GET |
| Marcar lido | `/api/v1/institutional/timeline/{id}/read` | POST |

## Gaps/ressalvas

- Virtual scrolling via ListView.builder default; medir fps em device low-end com 200+ items.
- "Marcar todos como lido" não no PDF; backlog M2.
