# M3 — Adicionar Vínculo (Mobile)

## Finalidade

Entry point para morador já cadastrado adicionar **outro condomínio** ao perfil. Bridge para o fluxo de onboarding parcial (ONB-M2..ONB-M5) reutilizado.

## Persona e ABAC

- **Persona**: Morador autenticado.
- **ABAC action**: `membership.create.self`.

## Fluxo mobile

1. Entrada: botão "Adicionar outro condomínio" de M2, empty state de M1, ou menu Perfil.
2. Mostra aviso: "Você pode ter múltiplos vínculos ativos. Cada avaliação e Connect Me é por condomínio."
3. CTA "Começar" → navega para [[../onboarding/ONB-M2]] com flag `mode=add_binding`.
4. Finaliza em ONB-MFIN → volta para M1 com novo condomínio ativo.

## Widget Flutter principal

```dart
class M3AdicionarVinculoPage extends StatelessWidget {
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Adicionar condomínio')),
      body: const _InstructionBody(),
      bottomNavigationBar: _CtaButton(
        onPressed: () => context.push('/onboarding/resident/m2?mode=add_binding'),
      ),
    );
  }
}
```

## Platform-specific notes

- **Back button Android**: volta para origem (M1 ou M2) sem resetar onboarding.
- **iOS**: swipe-back habilitado.
- **Both**: SafeArea; CTA fixo na base.

## Estados

- **Default**: texto + imagem ilustrativa + CTA.
- **Sem estados remotos** (navegação pura).

## Offline behavior

- CTA desabilitado se offline (busca CEP em ONB-M2 exige rede).
- Banner "Recurso exige conexão".

## Push notification trigger

Nenhum.

## Gaps/ressalvas

- Se usuário já tem N memberships em plano free → bloquear com modal upgrade? Decisão plano D-057 pendente.
