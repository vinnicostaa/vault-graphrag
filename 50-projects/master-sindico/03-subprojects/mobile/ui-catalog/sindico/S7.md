# S7 — Timeline Read Síndico (Mobile)

## Finalidade

Equivalente ao M6 do morador, **com metadados extras** exclusivos do síndico: autor, audit ref, hash integridade. Read-only no mobile.

## Persona e ABAC

- Síndico, `timeline.read.privileged`.

## Fluxo mobile

1. Chegada via card S6 "Linha do Tempo".
2. `GET /institutional/timeline/{cond}` (mesmo endpoint) + flag `?scope=syndic` para campos extras.
3. Cada item: chip categoria + título + preview + **autor** + **audit_ref**.
4. Tap → detail síndico (expande metadata).

## Widget Flutter principal

Reutiliza `TimelinePage` do morador com `variant: TimelineVariant.syndic` alterando `TimelineItemTile` para exibir meta linha.

## Platform-specific notes

Idênticas ao M6; apenas layout item ligeiramente diferente.

## Estados

Mesmos de M6.

## Offline behavior

Cache Hive `timeline_syndic_{cond}`.

## Push notification trigger

- `timeline.{cond}.new` mesmo do morador.

## Gaps/ressalvas

- Publicação/edição = web; mobile síndico não cria entrada em M1.
- Audit ref clickable abre detalhe de audit log (M3 apenas).
