# S2 — Seletor de Condomínios (Mobile)

## Finalidade

Lista condomínios onde o usuário tem role `syndic` ativo. Seleção define contexto de todas as telas seguintes.

## Persona e ABAC

- Síndico.
- `membership.self.read?role=syndic`.

## Fluxo mobile

1. `GET /institutional/memberships/me?role=syndic` retorna lista.
2. Cada item: nome condomínio + endereço curto + chip role_type (`principal/auxiliar/assistente`).
3. Tap → persiste default + navega `S6`.
4. Pull-to-refresh.

## Widget Flutter principal

```dart
class SeletorCondominiosPage extends StatelessWidget {
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('Escolha o condomínio')),
      body: BlocBuilder<SindicoMembershipsCubit, SindicoMembershipsState>(
        builder: (_, s) => RefreshIndicator(
          onRefresh: () async => ctx.read<SindicoMembershipsCubit>().load(),
          child: ListView.separated(...),
        ),
      ),
    );
  }
}
```

## Platform-specific notes

- Se apenas 1 condomínio → auto-redirect S6 (sem mostrar lista).
- iOS: haptic light ao selecionar.

## Estados

- Loading / Loaded / Empty ("Você ainda não foi atribuído como síndico") / Error / Offline.

## Offline behavior

- Cache Hive `syndic_memberships` 1h.
- Seleção offline OK (apenas altera contexto local); fetch dashboard em S6 vai falhar → banner.

## Push notification trigger

- `syndic.membership.added.{user_id}` → stale + refetch.

## Gaps/ressalvas

- Cliente não documenta limite de condomínios por síndico; assumido ilimitado.
