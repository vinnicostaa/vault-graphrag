# S1 — Entrada da Governança (Mobile)

## Finalidade

Tela institucional de boas-vindas exibida apenas no **primeiro acesso** do síndico à área Governança via mobile. Registra aceite formal de termos.

## Persona e ABAC

- Síndico (role `syndic`).
- `governance.entry.read`.

## Fluxo mobile

1. Após login, `AuthBloc` navega para `S2`; antes, checa `GET /syndic/governance/first-access`.
2. Se `first_access=true` → redirect `S1`.
3. Mensagem institucional + botão único "Entrar na Governança".
4. `POST /syndic/governance/acceptance` registra timestamp → navega S2.

## Widget Flutter principal

```dart
class GovernancaEntradaPage extends StatelessWidget {
  Widget build(BuildContext context) {
    return BlocConsumer<GovernanceEntryBloc, GovernanceEntryState>(
      listener: (ctx, s) => s.whenOrNull(accepted: () => ctx.go('/syndic/select')),
      builder: (_, s) => Scaffold(
        body: SafeArea(child: _Hero()),
        bottomNavigationBar: _AcceptButton(),
      ),
    );
  }
}
```

- `_Hero`: título grande + tagline + texto institucional.
- `_AcceptButton`: `FilledButton` full width altura 56dp.

## Platform-specific notes

- iOS: swipe-back **desabilitado** (tela gate).
- Android: back button = `SystemNavigator.pop()` (não permite bypass).
- **SafeArea** full + scroll vertical se texto longo.

## Estados

- Loading (checar flag) / Ready / Accepting / Accepted (navigate) / Error.

## Offline behavior

- **Online obrigatório** — aceite precisa timestamp server-side.
- Banner "Conecte-se para continuar".

## Push notification trigger

Nenhum.

## Integração backend

| Ação | Endpoint | Método |
|---|---|---|
| Verificar first_access | `/api/v1/syndic/governance/first-access` | GET |
| Registrar aceite | `/api/v1/syndic/governance/acceptance` | POST |

## Gaps/ressalvas

- Texto legal versionado — `terms_version` vem do servidor.
- Aceite grava IP/user-agent/device_fingerprint para compliance.
