# S6 — Central de Governança (Mobile)

## Finalidade

Hub principal do síndico mobile. Grade de cards read-only com navegação para timeline, plano diretor, eventos, comunicados, compliance, connect-me (M2), indicadores (M3).

## Persona e ABAC

- Síndico, `governance.dashboard.read`.

## Fluxo mobile

1. Ao selecionar condomínio em S2 → navega S6.
2. `GET /syndic/dashboard/{cond_id}` retorna cards + counters.
3. Grid 2x3 cards (responsivo — 2x4 em tablets).
4. Tap card → feature correspondente.

## Widget Flutter principal

```dart
class CentralGovernancaPage extends StatelessWidget {
  Widget build(BuildContext context) {
    return BlocBuilder<SindicoHomeBloc, SindicoHomeState>(
      builder: (_, s) => s.maybeWhen(
        ready: (dash, compliance) => _Grid(dash, compliance),
        loading: () => _SkeletonGrid(),
        offline: (stale) => _GridWithOfflineBanner(stale),
        orElse: () => _Error(),
      ),
    );
  }
}
```

- Cards: 6-8 dinâmicos conforme plano ativo + ABAC.

## Platform-specific notes

- Grid responsivo via `LayoutBuilder` + `GridView.count` com `crossAxisCount` calculado.
- Pull-to-refresh.
- SafeArea top.

## Estados

- Loading / Ready / Offline / Error.

## Offline behavior

- Cache Hive `syndic_dashboard_{cond_id}` 10min.
- Escrita inexistente em M1 → offline OK.

## Push notification trigger

- `connect-me.new.{cond_id}` → badge card Connect Me (M2).
- `compliance.alert.{cond_id}` → badge card Compliance.

## Componentes

- `GovernanceCard(icon, title, badge?, ctaRoute)`.

## Gaps/ressalvas

- Quais cards aparecem depende de ABAC + plano; ainda não formalizado.
- "Modo apresentação" (M3) seria aqui mas por enquanto fora.
