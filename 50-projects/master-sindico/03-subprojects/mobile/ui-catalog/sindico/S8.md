# S8 — Compliance View (Mobile)

## Finalidade

Visão read-only do compliance do condomínio: score atual + lista de alertas C1..C12 + status (ok/warn/critical). Resolução = web.

## Persona e ABAC

- Síndico, `compliance.self.read`.

## Fluxo mobile

1. Chegada via card S6 "Compliance".
2. `GET /compliance/{cond_id}/summary` retorna score + checks.
3. Score em destaque (gauge visual); abaixo, lista checks agrupada por severidade.
4. Tap check → expand com descrição + CTA "Resolver no web".

## Widget Flutter principal

```dart
class ComplianceViewPage extends StatelessWidget {
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('Compliance')),
      body: BlocBuilder<ComplianceCubit, ComplianceState>(
        builder: (_, s) => s.maybeWhen(
          loaded: (summary) => CustomScrollView(slivers: [
            _SliverScoreHeader(summary.score),
            _SliverChecksList(summary.checks),
          ]),
          orElse: () => _Skeleton(),
        ),
      ),
    );
  }
}
```

- Score gauge: `CustomPaint` ou `syncfusion_flutter_gauges`.
- Checks: `ExpansionTile` por item.

## Platform-specific notes

- Cores severity: usar palette Manual IV (OKLCH).
- Haptic warning ao expandir check crítico.

## Estados

- Loading / Loaded / Error / Offline.

## Offline behavior

- Cache Hive `compliance_{cond}` 30min.

## Push notification trigger

- `compliance.alert.{cond_id}.critical` → canal `ms_urgent`.

## Gaps/ressalvas

- Score calculation = server-side; mobile só exibe.
- "Resolver" abre web em Custom Tabs (Android) / SFSafariViewController (iOS).
