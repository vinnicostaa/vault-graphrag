# Cloudflare R2

**Categoria**: Object storage S3-compatible.

**Intenção**: armazenamento de blobs (imagens, vídeos, uploads, backups, artifacts, CDN origin) com preço baixo de armazenamento **e zero fee de egress** — principal diferenciador vs AWS S3.

## Quando usar

1. **Assets de CDN** (imagens, vídeos, fonts, CSS/JS buildado) — combina com CDN default ou Cache Rules.
2. **User uploads** (avatars, anexos, arquivos de usuário).
3. **Backups / snapshots** — barato para manter e barato para restaurar (zero egress).
4. **Data lake / analytics frios** quando acesso é via Workers.
5. **Migration de S3** — APIs S3 compatíveis, ferramentas `aws s3` funcionam com endpoint swap.

## Quando **NÃO** usar

- **Database-as-a-service** — não substitui D1/Postgres; é blob.
- **Streaming de vídeo adaptativo (HLS/DASH)** — [[../mux/_moc|Mux]] ou [[_moc|Stream (lazy)]] fazem transcoding + ABR; R2 apenas armazena arquivo cru.
- **Metadata pesado / queries de objeto** — S3 tem Metadata Search + S3 Inventory; R2 é mais simples. Pra metadata queryable, complementar com D1/KV.

## Padrão canônico

### Binding no Worker

```toml
# wrangler.toml
[[r2_buckets]]
binding = "UPLOADS"
bucket_name = "my-app-uploads"
```

```ts
interface Env { UPLOADS: R2Bucket }

// Upload
await env.UPLOADS.put('path/to/file.jpg', request.body, {
  httpMetadata: { contentType: 'image/jpeg' },
  customMetadata: { uploaderId, timestamp: String(Date.now()) }
})

// Read
const obj = await env.UPLOADS.get('path/to/file.jpg')
if (!obj) return new Response('not found', { status: 404 })
return new Response(obj.body, {
  headers: { 'Content-Type': obj.httpMetadata?.contentType ?? 'application/octet-stream' }
})

// Delete
await env.UPLOADS.delete('path/to/file.jpg')

// List
const list = await env.UPLOADS.list({ prefix: 'uploads/user-123/', limit: 100 })
for (const obj of list.objects) { /* ... */ }
```

### Acesso S3 externo (CLI, SDKs)

R2 expõe endpoint S3-compatible:

```bash
# ~/.aws/credentials
[r2]
aws_access_key_id = <R2_ACCESS_KEY>
aws_secret_access_key = <R2_SECRET_KEY>

aws s3 ls s3://my-app-uploads/ \
  --endpoint-url https://<accountid>.r2.cloudflarestorage.com \
  --profile r2
```

Criar credentials: dashboard → R2 → Manage R2 API Tokens.

### Signed URLs (presigned)

```ts
// Via API (account-level — não binding)
// aws4fetch é a lib standard em Workers
import { AwsClient } from 'aws4fetch'

const client = new AwsClient({
  accessKeyId: env.R2_ACCESS_KEY_ID,
  secretAccessKey: env.R2_SECRET_ACCESS_KEY,
})

const url = new URL(`https://${accountId}.r2.cloudflarestorage.com/my-app-uploads/${key}`)
url.searchParams.set('X-Amz-Expires', '3600')

const signed = await client.sign(new Request(url, { method: 'PUT' }), {
  aws: { signQuery: true }
})

return Response.json({ uploadUrl: signed.url })
```

Cliente faz PUT direto no bucket — não passa pelo Worker (ver [[patterns]] §5).

## Limites (consultada 2026-04-24)

| Limite | Valor |
|---|---|
| Tamanho máximo de objeto | 5 TB (por upload multipart) |
| Upload single-part | 5 GB |
| Bucket por conta | 1000 |
| Object key length | 1024 bytes |
| Buckets jurisdiction locked | EU, FedRAMP (via enterprise) |

## Preço (modelo principal)

| Dimensão | Custo |
|---|---|
| **Storage** | $0.015 / GB-mo |
| **Class A ops** (write: put, copy, post, list) | $4.50 / 1M |
| **Class B ops** (read: get, head) | $0.36 / 1M |
| **Egress** | **$0** (zero fees) |
| Free tier | 10 GB + 1M Class A + 10M Class B + zero egress |

**Comparação com S3** (us-east-1, para contexto):
- S3 storage: $0.023/GB-mo (1.5× mais caro que R2)
- S3 egress: $0.09/GB nos primeiros 10TB (R2 = $0)
- Exemplo real: 10TB storage + 50TB egress/mês — S3 ≈ $4,730/mo; R2 ≈ $150/mo.

## Antipatterns específicos

- **Servir vídeo longo direto do R2 via Worker** — Worker CPU time (30s paid) + stream é custoso. Ver [[antipatterns]] §3; usar [[_moc|Stream (lazy)]] (lazy) ou Mux.
- **Muitos objetos pequenos** — cada object é pelo menos 1 Class A write. Agrupe (tarballs, formato colunar) se >1M arquivos de <1KB.
- **List recursiva em bucket gigante** — `list()` retorna até 1000 por chamada; usar `cursor` para paginação. Em buckets >1M, manter index próprio em D1/KV.
- **Assumir "zero egress" para todas operações** — egress é zero; mas reads ainda pagam Class B ops ($0.36/1M). 1M views de imagem = $0.36 ainda.
- **Esquecer CORS em bucket público** — signed URL funciona, mas fetch direto pelo browser precisa CORS configurado (`wrangler r2 bucket cors put my-bucket`).

## Relações

- **Combina com**: [[workers]] (acesso via binding), [[pages]] (assets grandes fora do build output), [[_moc|CDN (lazy)]] (lazy — cachear frequente)
- **Alternativas**: AWS S3 (quando egress pattern diferente); GCS; Backblaze B2
- **Not-a-replacement**: [[d1]] (relacional), [[kv]] (key-value pequeno), [[_moc|Stream (lazy)]] (video transcoding + ABR)
