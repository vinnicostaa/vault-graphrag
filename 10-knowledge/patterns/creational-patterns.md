# Creational Patterns (categoria GoF)

Padrões criacionais provêem **mecanismos variados de criação de objetos**, aumentando flexibilidade e reuso do código existente. Os 5 padrões clássicos da categoria GoF:

- [[factory-method]] — interface para criar objetos numa superclasse, com subclasses decidindo o tipo concreto.
- [[abstract-factory]] — produz famílias de objetos relacionados sem acoplar ao tipo concreto.
- [[builder]] — constrói objetos complexos passo a passo; o mesmo código de construção gera variantes do objeto.
- [[prototype]] — copia objetos existentes sem acoplar o código ao tipo concreto deles.
- [[singleton]] — garante instância única de uma classe + ponto global de acesso.
